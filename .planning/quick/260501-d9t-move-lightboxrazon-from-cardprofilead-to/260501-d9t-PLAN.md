<objective>
Move LightboxRazon out of the repeating CardProfileAd component and mount it once at the global layout level (apps/website/app/layouts/default.vue), following the exact pattern used by LightboxLogin / LightboxSearch (Pinia app.store flag + self-contained component, no props from parents).

Purpose: When LightboxRazon renders inside CardProfileAd it inherits the card's CSS / stacking context, which causes visual and z-index issues (one lightbox per card, backdrop scoped to card, etc.). Hoisting it to the layout matches every other lightbox in the website and yields a single, correctly-stacked overlay.

Output: A single global <LightboxRazon /> mount controlled by useAppStore().openDeactivateLightbox(documentId) — CardProfileAd becomes a thin trigger.
</objective>
