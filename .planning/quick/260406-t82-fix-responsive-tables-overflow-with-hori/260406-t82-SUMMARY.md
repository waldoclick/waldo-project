# Quick Task 260406-t82 — Summary

**Task:** fix responsive: tables overflow with horizontal scroll, mobile menu overlaps content  
**Date:** 2026-04-07

## What was done

Two targeted CSS fixes in `apps/dashboard/app/scss/components/_layout.scss`:

### Fix 1: Table horizontal overflow at page level
Added `min-width: 0` to `.layout--dashboard__content`.

**Root cause:** CSS flexbox flex items have `min-width: auto` by default. Without `min-width: 0`, the content flex item expanded to fit wide table content (cells with `white-space: nowrap`) instead of staying bounded to the viewport. This caused the entire page to scroll horizontally, bypassing the `overflow-x: auto` on table wrappers.

**Fix:** `min-width: 0` forces the flex item to shrink to its container boundary. Table wrappers now correctly contain their own horizontal scroll.

### Fix 2: Mobile bottom nav overlapping content
Changed `position: absolute` → `position: fixed` on `.layout--dashboard__mobile` at `@include screen-small`.

**Root cause:** `position: absolute` positions relative to the document (scrolls with page). When page content was taller than the viewport, the mobile bottom bar scrolled away and appeared to overlap content at scroll positions. `position: fixed` keeps it permanently at the viewport bottom.
