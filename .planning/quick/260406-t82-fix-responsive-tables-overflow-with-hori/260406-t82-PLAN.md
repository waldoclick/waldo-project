# Quick Task 260406-t82: Fix Responsive — Tables & Mobile Menu

## Root Causes

1. **Tables causing full-page horizontal scroll**: `.layout--dashboard__content` is a flex item with `flex: 1` but no `min-width: 0`. CSS flexbox default is `min-width: auto`, meaning the item expands to fit its content instead of shrinking. This causes any wide table (with `white-space: nowrap` on cells) to overflow the viewport at the body level instead of staying contained in the table-wrapper's `overflow-x: auto`.

2. **Mobile menu overlapping and not staying fixed**: `.layout--dashboard__mobile` used `position: absolute` at `@include screen-small`. This positions it relative to the document, not the viewport. When the page content is taller than the viewport, scrolling causes the mobile bar to scroll away and overlap content. Needs `position: fixed` to stay at viewport bottom.

## Tasks

### Task 1: Fix table horizontal scroll — add min-width: 0 to layout content
- **File**: `apps/dashboard/app/scss/components/_layout.scss`
- **Change**: Add `min-width: 0` to `.layout--dashboard__content`
- **Status**: DONE

### Task 2: Fix mobile menu — change position absolute to fixed
- **File**: `apps/dashboard/app/scss/components/_layout.scss`
- **Change**: Change `position: absolute` → `position: fixed` on `.layout--dashboard__mobile` in `@include screen-small`
- **Status**: DONE
