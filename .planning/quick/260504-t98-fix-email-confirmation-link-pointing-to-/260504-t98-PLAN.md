# Quick Task 260504-t98: Fix Email Confirmation URL

## Context

Email confirmation links currently point directly to Strapi API:
`http://localhost:1337/api/auth/email-confirmation?confirmation=TOKEN`

Should instead point to frontend:
`http://localhost:3000/registro/activar?confirmation=TOKEN`

The frontend page then calls the API via Nuxt proxy (server/api/[...].ts).

## Task 1: Fix confirmationUrl in Strapi

**Files:** `apps/strapi/src/extensions/users-permissions/controllers/authController.ts`

**Action:**
1. Fix `confirmationUrl` in `registerUserLocal` (line ~197):
   - From: `${process.env.APP_URL || "http://localhost:1337"}/api/auth/email-confirmation?confirmation=${token}`
   - To: `${process.env.FRONTEND_URL || "http://localhost:3000"}/registro/activar?confirmation=${token}`

2. Fix `confirmationUrl` in `overrideSendEmailConfirmation` (line ~556):
   - Same replacement

3. Add `overrideEmailConfirmation` function at the end of the file:
   - Reads `ctx.query.confirmation` token
   - Finds user by `confirmationToken`
   - Returns 400 if token invalid/user not found
   - Returns `{ ok: true }` if already confirmed
   - Sets `confirmed: true` via `strapi.db.query` update
   - Returns `{ ok: true }` on success

**Verify:** Both confirmationUrl strings use FRONTEND_URL. overrideEmailConfirmation function exists and exported.
