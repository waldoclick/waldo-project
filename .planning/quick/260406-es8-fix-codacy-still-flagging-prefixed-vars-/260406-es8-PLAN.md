In `.eslintrc.json`, update the `rules` section:

1. Change `"no-unused-vars"` from `["warn", {...}]` to `"off"` — the base ESLint rule must be disabled when using the TS-ESLint equivalent (standard recommendation from typescript-eslint docs, prevents double-reporting).

2. Add `"@typescript-eslint/no-unused-vars"` with the same ignore patterns that were on the base rule:
```json
"@typescript-eslint/no-unused-vars": ["warn", {
  "argsIgnorePattern": "^_",
  "varsIgnorePattern": "^_",
  "caughtErrorsIgnorePattern": "^_",
  "destructuredArrayIgnorePattern": "^_",
  "ignoreRestSiblings": true
}]
```

This is the key fix: `plugin:@typescript-eslint/recommended` already enables `@typescript-eslint/no-unused-vars` (as "warn") but WITHOUT any ignore patterns. By explicitly configuring it in rules, our patterns override the preset defaults. The base `no-unused-vars` must be off because both rules checking the same code causes conflicts (TS-ESLint rule understands TypeScript constructs like constructor parameter properties that the base rule does not).

Keep all other rules unchanged. Keep the JSON comments for `no-undef` and `vue/multi-word-component-names`.
