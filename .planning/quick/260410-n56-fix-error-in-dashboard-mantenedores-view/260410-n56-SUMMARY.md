# Phase quick-260410-n56 Plan 01: Fix Error in Dashboard Mantenedores View Summary

**One-liner:** Fixed broken navigation in all 8 mantenedor sections by adding documentId to interfaces, switching router.push to use /maintenance/{entity}/{documentId} paths, and fixing post-save redirects and breadcrumbs to include the /maintenance/ prefix.

| 3 | Fix edit page breadcrumbs — add /maintenance/ prefix to all 8 edit pages | 4df97e90 |

## What Was Fixed

**Three classes of bugs across all 8 mantenedor sections (communes, regions, conditions, categories, faqs, packs, policies, terms):**

1. **Default table components**: View/Edit button click handlers accepted `number` id and routed to `/{entity}/{numericId}` — landing on 404. Fixed to accept `documentId: string` and route to `/maintenance/{entity}/{documentId}`.

2. **Form components**: Post-save `router.push` after PUT/POST was routing to `/{entity}/{id}` without the `/maintenance/` prefix — landing on 404 after creating or updating. Fixed to use `/maintenance/{entity}/{id}`.

3. **Edit page breadcrumbs**: Both the list breadcrumb (e.g. `to: "/communes"`) and the detail breadcrumb (e.g. `` to: `/communes/${id}` ``) were missing the `/maintenance/` prefix. Fixed to use `/maintenance/communes` and `/maintenance/communes/${id}`.

## Deviations from Plan

**None — plan executed exactly as written.**

Side note: `apps/dashboard/app/types/category.ts` and `apps/dashboard/app/types/pack.ts` were also updated to add `documentId: string` to their exported interfaces, since `CategoriesDefault.vue` and `PacksDefault.vue` import those types. This was required for TypeScript correctness and was implied by the plan's action description.
