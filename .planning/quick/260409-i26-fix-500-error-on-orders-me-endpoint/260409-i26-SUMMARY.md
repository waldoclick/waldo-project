# Quick Task 260409-i26: fix 500 error on orders/me endpoint

**Date:** 2026-04-09
**Commit:** 28b4565d

## Root cause

`mis-ordenes.vue` calls `loadUserOrders` with `sort: ["createdAt:desc"]` (array of strings). `@nuxtjs/strapi` serializes this as `sort[0]=createdAt:desc`, which Koa/qs parses back as `{ '0': 'createdAt:desc' }` (array-as-object). The `me` method passed this directly to `db.query orderBy` without normalization — `db.query` only accepts `{ field: 'asc'|'desc' }` objects, causing the 500.

The `find` method had already received this fix (commits 798c69a9, a13c337b) but `me` was missed.

## Fix

`apps/strapi/src/api/order/controllers/order.ts` — `me` method:

- Added sort normalization: handles string (`"createdAt:desc"`), string array (`["createdAt:desc"]`), and qs array-as-object (`{ '0': 'createdAt:desc' }`) — all converted to `{ createdAt: "desc" }` for `db.query`
- Added full populate normalization: same logic as `find` (handles `"*"`, arrays, array-as-objects)
- Replaced `ctx.internalServerError("Internal server error")` with `ctx.throw(500, error)` so actual error details surface in logs
