  Systematic diagnosis — run each check and record findings before making changes:

  1. **Check if OTHER NuxtImg components work**: Start dev server (`yarn dev` in apps/website), visit a page with LogoWhite or EmptyState. Open browser DevTools Network tab — do those NuxtImg images load? This tells us if IPX is globally broken or specific to this image/component.

  2. **Inspect the generated HTML**: Visit the home page, inspect the NuxtImg output in DevTools Elements tab. Check what `src` attribute NuxtImg generates — it may be something like `/_ipx/w_1200&h_600/images/bg-hero-home.webp`. Try loading that URL directly in the browser.

  3. **Check IPX endpoint**: Try `curl http://localhost:3000/_ipx/f_webp/images/bg-hero-home.webp` — does IPX respond with the image or an error? Try without format: `curl http://localhost:3000/_ipx/_/images/bg-hero-home.webp`.

  4. **Check terminal for errors**: Look at the Nuxt dev server terminal output for any IPX errors when the page loads (sharp errors, file not found, etc.).

  5. **Test NuxtImg without dimensions**: Remove `width` and `height` props from the NuxtImg to see if IPX chokes on resize parameters for webp. Test with just:
     ```vue
     <NuxtImg src="/images/bg-hero-home.webp" alt="..." loading="eager" />
     ```

  6. **Test with a known-working image format**: Copy the webp to a test jpg/png and try NuxtImg with that — isolates whether IPX has a webp processing issue with sharp.

  Based on diagnosis findings, apply the fix. Common fixes to try in order:
  - If IPX endpoint 404s or errors: check `image.dir` config (should point to `public/` or be default)
  - If IPX cannot process webp with dimensions: remove width/height or use `densities="1x"` to prevent upscaling
  - If IPX works but image is 0-sized in DOM: the issue is CSS — NuxtImg renders an `<img>` tag that must match the SCSS selector `.hero--home__bg img`
  - If sharp fails to process: add `image: { provider: "ipx", ipx: { sharp: {} } }` or try `provider: "none"` as last resort (still uses NuxtImg component but bypasses processing)

  IMPORTANT: The SCSS at `.hero--home__bg img` uses `position: absolute; width: 100%; height: 100%; object-fit: cover` — NuxtImg renders a plain `<img>` tag so this selector WILL match. Verify in DevTools that the rendered img element has these styles applied.
