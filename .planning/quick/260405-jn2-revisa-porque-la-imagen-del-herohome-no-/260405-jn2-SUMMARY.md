# Phase quick-260405-jn2 Plan 01: Fix Hero Background Image via NuxtImg + IPX

**One-liner:** Rebuilt IPX bundled sharp native binary (missing in monorepo) so NuxtImg serves hero background through /_ipx/ pipeline with 200 OK.

## Objective

Fix the hero home background image not rendering when using NuxtImg. Root cause: IPX bundled its own `sharp@0.32.6` at `node_modules/ipx/node_modules/sharp/` but the native binary `build/Release/sharp-linux-x64.node` was never built (Yarn hoisting skips install scripts for nested packages).

## Root Cause Analysis

**Symptom:** `/_ipx/_/images/bg-hero-home.webp` returns 500. PictureDefault renders NuxtImg but image is invisible.

**Error:** 
```
IPX_ERROR: Cannot find module '../build/Release/sharp-linux-x64.node'
Require stack: node_modules/ipx/node_modules/sharp/lib/sharp.js
```

**Why it happens:** IPX v2 bundles its own `sharp@0.32.6` as a local dependency inside `node_modules/ipx/node_modules/sharp/`. When Yarn installs the monorepo from root, it hoists packages and does not run `npm install` inside nested `node_modules` directories — so `sharp`'s native build step (`node-gyp rebuild`) never runs. The root `sharp@0.33.5` is built correctly, but IPX cannot use it because it requires its own bundled version.

**Fix applied:**
1. Manually ran `npm install --ignore-scripts=false` in `node_modules/ipx/node_modules/sharp/` to build the native binary
2. Added root `postinstall` script to `package.json` to repeat this automatically after every `yarn install`
3. Restarted Nuxt dev server to pick up the rebuilt binary

**Verification:**
- `/_ipx/_/images/bg-hero-home.webp` → 200 OK
- `/_ipx/s_1200x600/images/bg-hero-home.webp` → 200 OK (actual URL generated by NuxtImg)
- Homepage HTML: `src="/_ipx/s_1200x600/images/bg-hero-home.webp"` — NuxtImg generating correct IPX URL

## Component State After Fix

`PictureDefault.vue` uses `NuxtImg` (not `NuxtPicture`):
- No `format` prop — IPX handles format selection automatically
- No `<script>` block — `useImage()` preload was removed (caused issues, NuxtImg handles preload via `loading="eager"`)
- `width="1200" height="600"` remain for aspect ratio hints to IPX

SCSS selectors `hero--home__bg img` and `hero--default__bg img` match the `<img>` tag rendered by NuxtImg (confirmed: `NuxtImg` renders a plain `<img>` element, not `<picture><img>`).
