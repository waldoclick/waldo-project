# Quick Task 260409-qg2: Fix Active Menu Panel Not Persisting on Page Refresh

## Root Cause

`activeMenu` in `dashboard.vue` was always initialized to `"default"` regardless of the
current route. On hard refresh, Vue re-mounts the layout and loses the panel selection.

## Fix

- `resolveActiveMenu(path)` — pure function that maps route prefix to panel name
- `activeMenu` initialized via `resolveActiveMenu(route.path)` at setup time
- `watch(route.path)` keeps panel in sync with any programmatic navigation that bypasses
  the rail buttons (e.g. external link to `/maintenance/categories`)

## Task

1. Update `apps/dashboard/app/layouts/dashboard.vue`:
   - Import `watch` from vue, `useRoute` from vue-router
   - Add `resolveActiveMenu` helper
   - Initialize `activeMenu` from current path
   - Add watch to sync panel on route change
