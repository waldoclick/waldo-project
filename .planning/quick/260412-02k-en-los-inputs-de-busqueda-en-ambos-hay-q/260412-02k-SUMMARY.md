# Phase quick-260412-02k Plan 01: Search Input Maxlength Summary

**One-liner:** HTML maxlength=100 on both search inputs plus defensive slice in website handleSubmit to cap the ?s= URL param.

## Objective

Prevent unbounded text from being submitted through search inputs in both apps. Enforces 100-character limit at the browser level and defensively in the URL builder.

## Changes Made

### Task 1 — Website SearchDefault.vue
- Added `maxlength="100"` to `<input v-model="form.query" name="query" type="text" ...>`
- Changed `s: form.value.query.toLowerCase() || null` to `s: form.value.query.slice(0, 100).toLowerCase() || null` in `handleSubmit`
- Dual-layer enforcement: browser blocks input at 100 chars; slice guarantees the URL param never exceeds 100 chars even if v-model is written programmatically

### Task 2 — Dashboard SearchDefault.vue
- Added `maxlength="100"` to `<input :value="modelValue" :placeholder="placeholder" ...>`
- Single addition covers all dashboard list views (orders, packs, users, ads, etc.) that consume this reusable component via v-model
