<objective>
Add a 100-character maxlength limit to both search inputs (website public search and dashboard reusable search) to prevent unbounded text from being submitted.

Purpose: Protect search endpoints from excessively long query strings, prevent URL bloat on /anuncios, align with the dual-layer validation pattern established for auth forms (quick 260411-m0h).
Output: Two updated Vue components with HTML maxlength enforcement plus a defensive slice in the website's handleSubmit.
</objective>

<interfaces>
Both search inputs are plain HTML <input> elements with v-model. Neither uses a yup schema, so enforcement is HTML-only. The website search has a free `handleSubmit` we can harden with a `.slice(0, 100)` before building the query string.

Website SearchDefault relevant lines:
```vue
<input
  v-model="form.query"
  name="query"
  type="text"
  placeholder="Escribe…"
/>
```
handleSubmit currently does: `s: form.value.query.toLowerCase() || null`

Dashboard SearchDefault relevant lines:
```vue
<input
  :value="modelValue"
  :placeholder="placeholder"
  class="search--default__input"
  @input="$emit('update:modelValue', ($event.target as HTMLInputElement).value)"
/>
```
</interfaces>
