# Phase quick-260409-h79 Plan 01: Subscription Dashboard Views Summary

Read-only dashboard views for subscription-pros and subscription-payments collections, mirroring the packs pattern with TypeScript types, table components, SCSS, settings store, and four pages.

## What Was Built

Two new dashboard sections fully wired to Strapi:

- `/subscription-pros` — paginated table of PRO subscriptions showing user email, card type + last 4, pending invoice badge, and creation date
- `/subscription-pros/:id` — detail page with card type, last 4, pending invoice, creation/update timestamps
- `/subscription-payments` — paginated table of subscription payments showing user email, amount, status badge, period range, charged date, and attempt count
- `/subscription-payments/:id` — detail page with all payment fields (parent/child buy orders, authorization code, response code, period dates, charged date, next attempt); payment_response JSON blob excluded

Both sections are fully read-only: no Agregar, no edit buttons.

### Design Decisions

**1. BadgeDefault variant fallback for payment status**
- **Found during:** Task 2
- **Issue:** BadgeDefault only exposes `"default" | "outline" | "secondary"` variants. The plan mentioned success/error/neutral as ideal but instructed to fall back to "outline" if not supported.
- **Fix:** `statusVariant` maps `approved -> "default"`, `failed -> "secondary"`, `deactivated -> "outline"`.
- **Files modified:** apps/dashboard/app/components/SubscriptionPaymentsDefault.vue

## Known Stubs

None. Both components fetch live data from Strapi via useApiClient with populate=user. Pages use useAsyncData with documentId filter and numeric id fallback.
