<objective>
Refactor `apps/website/app/pages/anuncios/[slug].vue` to use the correct Nuxt 4 error-propagation pattern: throw `createError({ statusCode: 404, fatal: true })` inside the `useAsyncData` callback when no ad is found, and throw `createError({ statusCode: 500, fatal: true })` on unexpected processing errors. Remove the `watchEffect` + `showError()` approach (which fires during SSR and causes 500s), and add `default: () => null` to the `useAsyncData` options.

Purpose: The `watchEffect`/`showError` pattern races SSR lifecycle ordering and calls `showError()` inside the SSR renderer instead of Nuxt's error boundary — producing 500s. Throwing `createError` inside `useAsyncData` is the only SSR-safe approach and the AGENTS.md-mandated pattern.
Output: `[slug].vue` refactored — `watchEffect`, `showError`, `getErrorMessage`, unused `pending`, unused `adError` all deleted; `createError` with `fatal: true` added at two null-return paths; `default: () => null` added to options.
</objective>
