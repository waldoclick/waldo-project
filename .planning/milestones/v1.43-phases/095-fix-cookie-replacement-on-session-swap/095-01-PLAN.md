<objective>
Fix the zombie-cookie bug that leaves a stale `waldo_jwt` cookie with the shared-domain attribute after a dashboard session replacement.

Purpose: When a website user with a `domain=.waldo.click` JWT visits the dashboard and logs in as manager, the existing cookie must be fully cleared before the new one is written. `existingCookie.value = null` clears only the host-only cookie — the shared-domain version survives, creating two competing `waldo_jwt` cookies. The browser uses the more specific (host-only) one, masking the new session.

Output:
- `FormLogin.vue` — uses `useStrapiAuth().logout()` which respects the COOKIE_DOMAIN attribute from `@nuxtjs/strapi` module config, clearing both cookie scopes
- `apps/website/nuxt.config.ts` + `apps/dashboard/nuxt.config.ts` — `ad_reservations.ad` and `ad_featured_reservations.ad` removed from `auth.populate` (dead joins, not in User TypeScript interface, same SSR-slowness root cause diagnosed in Phase 094)
</objective>

```typescript
const handleSubmit = async (values: Record<string, unknown>) => {
  // Check if there's an existing non-manager session cookie
  const existingCookie = useCookie("waldo_jwt");
  if (existingCookie.value) {
    const confirmed = await Swal.fire({
      title: "Ya tienes una sesión activa",
      text: "Estás conectado con un usuario que no es administrador. ¿Deseas cerrar esa sesión e iniciar sesión como administrador?",
      icon: "warning",
      showCancelButton: true,
      confirmButtonText: "Continuar",
      cancelButtonText: "Cancelar",
    });
    if (!confirmed.isConfirmed) return;
    // Clear the existing session cookie using strapiLogout() so the shared-domain cookie
    // is cleared correctly (respects domain attribute from nuxt.config.ts strapi.cookie)
    const { logout: strapiLogout } = useStrapiAuth();
    await strapiLogout();
  }
  // ... rest unchanged
```

**Do NOT use `useLogout()` composable** — that also resets Pinia stores (appStore, meStore, searchStore) and navigates to `/auth/login`, which would break the login flow mid-execution.

**Note:** `useStrapiAuth` is a Nuxt auto-import from `@nuxtjs/strapi` — no import statement needed at the top of the script block.
