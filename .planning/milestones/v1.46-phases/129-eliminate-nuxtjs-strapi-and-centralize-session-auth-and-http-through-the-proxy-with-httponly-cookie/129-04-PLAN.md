<objective>
Migrate the five Google/Facebook OAuth logic files to the httpOnly model: the popup no longer carries a jwt (the Nitro popup callback set the cookie); `LoginWithGoogle.vue` drops `setToken` and calls `fetchUser`; the redirect pages call the new `/api/auth/{provider}/exchange` routes (which set the cookie) through `useApiClient` (so the JWT-issuing POST carries `X-Recaptcha-Token`) instead of `authenticateProvider`; and the One Tap plugin drops `setToken` (its server route set the cookie).

NOTE — knowingly half-migrated until cutover: these files now use `useSessionAuth`/`useSessionUser`, but the `@nuxtjs/strapi` module remains the active session writer until plan 06 activates `session.ts`. End-to-end OAuth login is therefore only verified by the human-verify checkpoints in plan 06. Acceptance here is grep/structure-based.

Purpose: Remove all client-side token handling from the OAuth surface.
Output: 5 migrated files; zero setToken/authenticateProvider in them.
</objective>

  <action>
CRITICAL — reCAPTCHA: the exchange routes (plan 02 Task 3) are JWT-issuing POST routes that now call `verifyRecaptchaToken` against the `x-recaptcha-token` header. The caller MUST therefore send that header. Route the call through `useApiClient` (NOT raw `$fetch`) — `useApiClient` executes reCAPTCHA and injects `X-Recaptcha-Token` on POST automatically. A raw `$fetch("/api/auth/google/exchange", { method: "POST" })` would omit the token and the route would reject with 400.

CRITICAL — must run CLIENT-SIDE: `useApiClient`'s reCAPTCHA execution is client-only (`$recaptcha` is undefined on SSR, so no token would be sent). The exchange call MUST run on the client — keep it in the existing post-redirect handler (the same place `authenticateProvider` was called, e.g. inside `onMounted` / a client-side handler). Do NOT hoist it into `useAsyncData` or a top-level SSR `await` — that would send no `X-Recaptcha-Token` and the guarded route would reject. (Replacing `authenticateProvider` in place keeps it client-side, which is what we want.)

3. Replace the `authenticateProvider("google", ...)` call with a POST through `useApiClient` to the exchange route. Pass the Strapi-relative path `auth/google/exchange` (useApiClient prefixes `/api`, and the dedicated Nitro route matches before the catch-all):
