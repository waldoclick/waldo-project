<objective>
Migrate all CRUD admin form components and the me.store from `useStrapi()` SDK calls (`strapi.create`, `strapi.update`) to `useApiClient()` direct HTTP calls. These components manage FAQ, commune, region, category, pack, condition, and password data.

Purpose: The `useStrapi()` SDK methods (`strapi.create`, `strapi.update`, `strapi.delete`) do not support custom headers, so they cannot inject the `X-Recaptcha-Token`. After Plan 01 broadened the server guard, these calls will be rejected. They must be replaced with `useApiClient()` calls that construct equivalent HTTP requests.
Output: 8 files migrated, all CRUD admin operations now protected by reCAPTCHA.
</objective>

```typescript
export function useApiClient(): (url: string, options?: Parameters<typeof client>[1]) => Promise<T>
// Drop-in replacement for useStrapiClient()
// Auto-injects X-Recaptcha-Token on POST, PUT, DELETE
```

Migration patterns (from RESEARCH.md):
```typescript
// strapi.create("faqs", payload) → apiClient("/faqs", { method: "POST", body: { data: payload } })
// strapi.update("faqs", docId, payload) → apiClient(`/faqs/${docId}`, { method: "PUT", body: { data: payload } })
// strapi.delete("articles", docId) → apiClient(`/articles/${docId}`, { method: "DELETE" })
```

CRITICAL: `strapi.create` wraps payload as `{ data: payload }` automatically.
When migrating to `apiClient`, you MUST add `{ data: payload }` explicitly in the body.
