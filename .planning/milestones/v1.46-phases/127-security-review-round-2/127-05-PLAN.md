<objective>
Close SEC2-LOCKDOWN (four fixes):
1. MJML nunjucks is `autoescape: false` (`mjml/index.ts:4`); cron reports and transactional emails interpolate `ad.name`/`username`/`email` unescaped → stored XSS to admin/owner recipients.
2. Upload middleware (`upload.ts`) checks only the spoofable `mimetype` — no magic-byte verification, no size limit.
3. `GET /api/users` (`getUserDataWithFilters`) merges client `filters` into the `where` clause unsanitized and returns PII (email/phone/RUT/address).
4. `verification-code` (stores plaintext OTP code + pendingToken), `contact`, and `subscription-payment` use the bare core router → web exposure depends entirely on DB role grants.

Purpose: prevent email XSS, content-type-spoofed/oversized uploads, PII leakage and filter abuse on the users list, and code-level lockdown of sensitive content-API routes.
Output: autoescape with audited templates, file-type magic-byte + sizeLimit, filter whitelist + PII strip, disabled/limited core routes, and Jest regression tests.

NOTE: this plan is wave 2 because it edits `apps/strapi/config/plugins.ts`, which plan 127-03 also edits (users-permissions ratelimit). Apply your upload `sizeLimit` change on top of 03's changes — do not revert them.
</objective>
