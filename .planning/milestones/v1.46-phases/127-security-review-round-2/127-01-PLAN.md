<objective>
Close SEC2-PAYMENT: the Webpay return handler in `checkout.service.ts` (and the mirror in `pack.service.ts`) currently only checks `status === "AUTHORIZED"`. It never re-validates the paid amount, never protects against replay (the GET return URL stays in browser history and re-grants benefits), never verifies the ad belongs to the paying user, and silently falls back to `AD_FEATURED_PRICE || 10000`.

Purpose: prevent a user from paying less than the pack price, replaying the return URL to mint free credits/featured/publish, paying for an ad they do not own, or exploiting a missing price env var.
Output: amount re-validation, idempotency keyed on a now-unique `buy_order`, ad-ownership assertion at commit, fail-closed price, and Jest regression tests covering all four.
</objective>
