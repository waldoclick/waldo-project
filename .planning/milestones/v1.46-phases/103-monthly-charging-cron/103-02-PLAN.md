interface ProUser {
  id: number;
  documentId: string;

Export class `SubscriptionChargeService` with:

1. **`chargeExpiredSubscriptions(): Promise<ICronjobResult>`** — main entry point:
   - Validate `PRO_MONTHLY_PRICE` env var is set and non-zero; throw descriptive error if missing
   - Query users: `strapi.entityService.findMany("plugin::users-permissions.user", { filters: { pro_status: { $eq: "active" }, pro_expires_at: { $lte: today + "T23:59:59.999Z" } }, fields: ["id", "documentId", "tbk_user", "pro_expires_at"], pagination: { pageSize: -1 } })`
   - For each user: call `chargeUser(user, periodStart, today, 1)` where `periodStart` = `pro_expires_at` date portion (the start of the expired period)
   - Before calling chargeUser, run idempotency check: query `api::subscription-payment.subscription-payment` for `{ user: { id: user.id }, period_start: periodStart, status: "approved" }`. If found, skip.
   - Then process retries: query `api::subscription-payment.subscription-payment` with `{ status: "failed", charge_attempts: { $lt: 3 }, next_charge_attempt: { $lte: today } }`, populate `["user"]`. For each, call `chargeUser` with `attempt = record.charge_attempts + 1` and pass `record.id` (document ID) to update the existing record.
   - Then deactivate exhausted: query failed records with `charge_attempts >= 3`. For each, update user: `pro_status: "inactive"`, `pro: false`, `pro_expires_at: null`, `tbk_user: null`. Update record status to `"deactivated"`.
   - Wrap entire method in try/catch, return `ICronjobResult`.

2. **`chargeUser(user, periodStart, today, attempt, existingRecordId?): Promise<void>`** — private helper:
   - Build `parentBuyOrder = "pro-" + user.id + "-" + today.replace(/-/g, "")` (YYYYMMDD, no dashes)
   - Build `childBuyOrder = "c-" + user.id + "-" + today.replace(/-/g, "") + "-" + attempt`
   - Call `new OneclickService().authorizeCharge(user.documentId, user.tbk_user, amount, parentBuyOrder, childBuyOrder)`
   - On success (`result.success === true`):
     - If `existingRecordId`: update existing subscription-payment to `status: "approved"`, `authorization_code`, `response_code`, `payment_response: result.rawResponse`, `charged_at: new Date()`, increment `charge_attempts`
     - If new: create subscription-payment with `user: user.id`, `amount`, `status: "approved"`, `parent_buy_order`, `child_buy_order`, `authorization_code: result.authorizationCode`, `response_code: result.responseCode`, `payment_response: result.rawResponse`, `period_start: periodStart`, `charged_at: new Date()`, `charge_attempts: attempt`
     - Update user `pro_expires_at` to `new Date(Date.now() + 30 * 24 * 60 * 60 * 1000)`. Use the Strapi v5 cast pattern: `data as unknown as Parameters<typeof strapi.entityService.update>[2]["data"]`
   - On failure:
     - Compute `nextRetryDate`: if attempt=1, tomorrow; if attempt=2, +2 days (lands on day 3 after expiry)
     - If `existingRecordId`: update record with `charge_attempts: attempt`, `next_charge_attempt: nextRetryDate`, `response_code: result.responseCode`, `payment_response: result.rawResponse`
     - If new: create subscription-payment with `status: "failed"`, `charge_attempts: attempt`, `next_charge_attempt: nextRetryDate`, remaining fields from result
