<objective>
Replace all Strapi backend reads and writes of the `pro` boolean with `pro_status` enum checks. This is the server-side half of the migration: computeSortPriority, sanitizeAdForPublic, proResponse, subscription-charge cron, protect-user-fields middleware, and all associated type casts and tests.

Purpose: Eliminate the dual-source-of-truth between `pro` boolean and `pro_status` enum on the server side, preventing inconsistent state bugs.
Output: All Strapi backend code uses `pro_status === "active"` exclusively; no code path writes `pro: true/false`.
</objective>

```typescript
export function computeSortPriority(ad: {
  ad_featured_reservation?: unknown;
  user?: { pro_status?: string } | null;
}): number {
  const isFeatured =
    ad.ad_featured_reservation != null &&
    typeof ad.ad_featured_reservation === "object" &&
    "id" in (ad.ad_featured_reservation as Record<string, unknown>);
  if (!isFeatured) return 2;
  const isPro = !!(
    ad.user &&
    typeof ad.user === "object" &&
    (ad.user as { pro_status?: string }).pro_status === "active"
  );
  return isPro ? 0 : 1;
}
```
