# Phase 103.1 Plan 01: Remove pro boolean — Strapi backend Summary

**Eliminated dual-source-of-truth: all Strapi backend code now reads/writes pro_status exclusively, with no code path writing pro: true or pro: false**

## Accomplishments

- `computeSortPriority` signature and logic use `pro_status === "active"` exclusively — removed `pro?: boolean` everywhere
- `sanitizeAdForPublic` exposes `pro_status` instead of `pro` in the public API user object
- `proResponse` inscription handler no longer writes `pro: true` — `pro_status: "active"` is the sole field set
- Subscription deactivation cron no longer writes `pro: false` — `pro_status: "inactive"` is the sole field set

- Created `ad.sort-priority.test.ts` with 7 tests covering all `computeSortPriority` branches using `pro_status` mocks
