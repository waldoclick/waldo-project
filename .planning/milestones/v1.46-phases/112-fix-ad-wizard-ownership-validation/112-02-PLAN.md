Place it AFTER the `await useAsyncData("anunciar-init", ...)` call (after line ~54, after the closing `);` of useAsyncData). At this point `meStore.loadMe()` has resolved and `meStore.me` is guaranteed to be populated.

Add this block IMMEDIATELY AFTER the useAsyncData closing `);`:
```typescript
// Ownership guard: reset wizard if draft belongs to a different user (per SEC-112-01)
// Placed after useAsyncData so meStore.me is guaranteed populated by loadMe()
if (adStore.ad.ad_id && adStore.userId !== meStore.me?.id) {
  adStore.$reset();
}

// Track current user as draft owner for future ownership checks
adStore.userId = meStore.me?.id ?? null;
```

CRITICAL placement: This MUST be AFTER `await useAsyncData(...)` completes, NOT before it. The `meStore.loadMe()` call inside useAsyncData is what populates `meStore.me`. Placing the guard before useAsyncData would mean `meStore.me` is null, causing `$reset()` to fire for every legitimate user on every page load.

The `adStore.userId = meStore.me?.id ?? null` line runs on every wizard entry, ensuring the userId is always set to the current user. This covers both:
- New wizard sessions (userId was null, now set)
- Returning to an existing draft (userId re-confirmed)
