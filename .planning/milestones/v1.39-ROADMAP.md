**Status:** ✅ SHIPPED 2026-03-15

**Total Plans:** 7

**Timeline:** 2026-03-15 (1 day)

## Overview

Complete migration of the Waldo website's data-fetching layer from the `@nuxtjs/strapi` SDK to the `useApiClient` composable. All `strapi.find()` and `strapi.findOne()` calls replaced across 12 stores, 3 composables, and 5 pages/components. The Strapi SDK is now exclusively used for auth (`useStrapiAuth`) — all data reads and writes flow through `useApiClient`, which provides unified reCAPTCHA injection for mutations and clean GET passthrough.

## Milestone Summary

**Key Decisions:**

- `useApiClient()` placed at store root level (not inside action functions) — composable rules require setup-level instantiation
- Raw body shapes for collection `{ data: T[] }` and single-item `{ data: T }` endpoints are identical to Strapi SDK returns — callers of `useStrapi`/`useOrderById`/`usePacksList` required no changes
- `useApiClient()` inside factory function (not module level) for composables like `usePacksList` — same composable rules
- Store methods returning collection responses declare explicit return types when callers access `.data`/`.meta` — TypeScript infers `{}` from raw `client()` result without them
- `/api/` prefix removed from all `useApiClient` URLs after discovering `useStrapiClient` already applies it from config (post-ship fix commit `96653c4`)
- `FormProfile.vue` `useStrapi()` import was dead code — purely subtractive removal
- `anunciar/gracias.vue` response accessed via `response.data` (not `{ data: ad }` destructure) — `useApiClient` returns raw body

**Issues Resolved:**

- `strapi.find()` / `strapi.findOne()` calls had implicit `.data` wrapper behavior — every migrated call site required removing the `.data` accessor
- `categories.store.ts` was Options API store with per-action `useStrapi()` — migrated to single `useApiClient()` at setup root level
- Two TS2339 type errors in `user.store.ts` return signatures — fixed with explicit return type annotations in plan 090-06

**Issues Deferred:**

- Pre-existing Vitest failures in 6 unrelated test files (`useOrderById.test.ts`, `FormRegister.test.ts`, `FormLogin.website.test.ts`, `FormLogin.spec.ts`, `ResumeOrder.test.ts`, `recaptcha-proxy.test.ts`) — pre-date this milestone, not regression-introduced

**Technical Debt Incurred:**

- `apps/dashboard` still uses `@nuxtjs/strapi` SDK for data fetching — `apps/website` fully migrated; dashboard migration deferred to future milestone
