# v1.26 REQUIREMENTS: Mostrar comprobante Webpay en /pagar/gracias

## Goal
After a successful Webpay payment, users are redirected to /pagar/gracias with the order's documentId. The page uses this documentId to fetch order/payment information and displays it, focusing only on the order (not ad info).

## Requirements

- ORD-01: Upon successful Webpay payment, redirect to /pagar/gracias?order=<documentId> (no adId in query or state).
- ORD-02: /pagar/gracias page must accept the order's documentId as parameter/query.
- ORD-03: The page must fetch order/payment info by documentId, using the proper Strapi API (must not fetch or show ad info).
- ORD-04: The UI must only show order/payment summary: amount, status, Transbank receipt fields, timestamp, buyer name/email, and any additional payment metadata.
- ORD-05: All legacy/unused code referencing ad data or ad-centric display in /pagar/gracias must be removed.
- ORD-06: Automated test (unit or E2E) confirms correct fetch and display for a given documentId; edge cases (invalid/missing ID) covered with graceful fallback or error.

## Acceptance Criteria
- Successful payment always redirects with the correct order documentId.
- /pagar/gracias never fetches by adId; it works solely with the order documentId.
- Order/payment info is shown; no ad info visible.
- Fails gracefully (shows error/fallback UI) when documentId is missing or invalid.
- Code cleanup eliminates all ad-related handling in this page/flow.
