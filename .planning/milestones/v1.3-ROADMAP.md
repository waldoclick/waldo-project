# Milestone v1.3: Utility Extraction

**Status:** ✅ SHIPPED 2026-03-06

## Overview

Extract all inline duplicated pure functions (date, price, string) into shared utility files and replace every inline copy with an import — zero duplicate function definitions remain in the dashboard.

Delivered `app/utils/date.ts`, `app/utils/price.ts`, and `app/utils/string.ts` with full TypeScript strict typing, unit test coverage, and Nuxt auto-import integration. Eliminated 51 duplicated inline formatting definitions across the codebase.

## Milestone Summary

**Key Decisions:**
- Use `numeric: 'always'` for RelativeTimeFormat — ensures consistent 'hace X ...' formatting regardless of magnitude
- `Intl.NumberFormat('es-CL', { style: 'currency', currency: 'CLP' })` as canonical currency formatter
- Return `"--"` for all null/undefined/invalid inputs across all utilities — consistent UX for missing data
- Rename local formatCompactCurrency in ChartSales rather than removing — it serves a different purpose (compact notation)
- Nuxt auto-import picks up `app/utils/*.ts` automatically — no explicit imports needed in components/pages
- UTIL-07 (build validation) treated as a success criterion in every phase, not a standalone phase

**Issues Resolved:**
- 51 duplicated inline formatting function definitions eliminated across the codebase
- Inconsistent null handling for date/price/string display (some returned "", some crashed, some returned "N/A") — now unified to "--"
- ChartSales had a local `formatCurrency` that differed from the global one — resolved by explicit rename

**Issues Deferred:**
- None from this milestone

**Technical Debt Incurred:**
- None — all three utility files have 100% test coverage and strict TypeScript typing
