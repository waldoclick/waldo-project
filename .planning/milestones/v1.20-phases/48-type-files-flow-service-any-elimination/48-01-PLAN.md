## Context

Five files contain all the `any` types for this phase:

1. `apps/strapi/src/api/order/types/order.types.ts` — `QueryParams` has `filters?: any`, `sort?: any`, `populate?: any`; `Order` has `payment_response: any` and `document_details: any`
2. `apps/strapi/src/api/filter/types/filter.types.ts` — `StrapiFilter` has 12 operator fields (`$eq`, `$ne`, `$lt`, `$lte`, `$gt`, `$gte`, `$in`, `$nin`, `$contains`, `$notContains`, `$containsi`, `$notContainsi`) typed as `any`/`any[]`; `Category.icon?: any`
3. `apps/strapi/src/services/flow/factories/flow.factory.ts` — `type Strapi = any` local type alias
4. `apps/strapi/src/services/flow/types/flow.types.ts` — `IFlowSubscriptionResponse.discount?: any`, `invoices?: any[]`; `IFlowInvoice.items?: any[]`, `chargeAttemps?: any[]`; `IFlowPaymentStatusResponse.optional?: Record<string, any>`
5. `apps/strapi/src/services/flow/services/flow.service.ts` — `type Strapi = any` local alias; 6× `Record<string, any>` param bags (`signRequest` + 5 method `params` locals); 5× `(data as any).message` casts in Axios error-handling branches

No new files. No new packages. All changes are in-place edits.

`tsc --noEmit` passes with zero errors before this phase. It must continue to pass after.

The existing `flow.test.ts` has `const mockStrapi = {} as any` — this is **out of scope** per REQUIREMENTS.md and must not be modified.

**Step 4 — Fix all `(data as any).message` casts (TSANY-12):**

There are 5 locations in Axios error-handling branches inside catch blocks for: `createPaymentOrder`, `createSubscription`, `createCustomer`, `getInvoice`, `getCustomerSubscriptions`.

Also fix the 2 occurrences in `createPaymentOrder` that use `(responseData as any).message` in the validation block.

For **response validation block** in `createPaymentOrder` (lines 94-96):
```typescript
// BEFORE
} else if (
  typeof responseData === "object" &&
  responseData !== null &&
  (responseData as any).message
) {
  errorDetail = `Flow error message: ${(responseData as any).message}`;
}

// AFTER
} else if (
  typeof responseData === "object" &&
  responseData !== null
) {
  const responseObj = responseData as Record<string, unknown>;
  if (typeof responseObj.message === "string") {
    errorDetail = `Flow error message: ${responseObj.message}`;
  }
}
```

For **each Axios error catch block** with `(data as any).message` pattern (5 occurrences):
```typescript
// BEFORE
if (data) {
  detail = JSON.stringify(data);
  if (
    typeof data === "object" &&
    data !== null &&
    (data as any).message
  ) {
    detail = (data as any).message;
  }
}

// AFTER
if (data) {
  detail = JSON.stringify(data);
  if (typeof data === "object" && data !== null) {
    const dataObj = data as Record<string, unknown>;
    if (typeof dataObj.message === "string") {
      detail = dataObj.message;
    }
  }
}
```

Apply this pattern to the catch blocks in: `createPaymentOrder`, `createSubscription`, `createCustomer`, `getInvoice`, `getCustomerSubscriptions`.
