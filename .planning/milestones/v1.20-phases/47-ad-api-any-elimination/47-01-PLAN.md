## Task 47-01-A: Fix `any` types in `ad.ts` service

**File:** `apps/strapi/src/api/ad/services/ad.ts`
**Requirements:** TSANY-01, TSANY-02, TSANY-03, TSANY-04

### What to do

**Step 1 ‚Äî Add `AdQueryOptions` interface** at the top of the file (after the `AdStatus` type):

```typescript
interface AdQueryOptions {
  filters?: unknown;
  populate?: unknown;
  page?: string | number;
  pageSize?: string | number;
  sort?: unknown;
  orderBy?: unknown;
  pagination?: {
    page?: string;
    pageSize?: string;
  };
}
```

**Step 2 ‚Äî Fix `computeAdStatus` (TSANY-02):**
Change `ad: any` ‚Üí `ad: unknown` and add an object guard at the top:

```typescript
function computeAdStatus(ad: unknown): AdStatus {
  if (!ad || typeof ad !== "object") return "unknown";
  const adObj = ad as Record<string, unknown>;

  const hasReservationKey = Object.prototype.hasOwnProperty.call(adObj, "ad_reservation");

  if (adObj.rejected) return "rejected";
  if (adObj.banned) return "banned";
  if (adObj.active && !adObj.banned && !adObj.rejected && (adObj.remaining_days as number) > 0) return "active";
  if (!adObj.active && !adObj.banned && !adObj.rejected && adObj.remaining_days === 0) return "archived";
  if (
    !adObj.active && !adObj.banned && !adObj.rejected &&
    (adObj.remaining_days as number) > 0 &&
    hasReservationKey &&
    (adObj.ad_reservation == null || adObj.ad_reservation === undefined) &&
    adObj.is_paid
  ) return "abandoned";
  if (
    !adObj.active && !adObj.banned && !adObj.rejected &&
    (adObj.remaining_days as number) > 0 &&
    (!hasReservationKey || adObj.ad_reservation != null)
  ) return "pending";

  return "unknown";
}
```

**Step 3 ‚Äî Fix `transformSortParameter` (TSANY-03):**
Change `sort: any` ‚Üí `sort: unknown`, return type `any` ‚Üí `unknown`:

```typescript
function transformSortParameter(sort: unknown): unknown {
  if (!sort) return { createdAt: "desc" };
  if (typeof sort === "object" && !Array.isArray(sort)) return sort;
  if (typeof sort === "string") {
    const sortParts = sort.split(":");
    if (sortParts.length === 2) {
      const [field, direction] = sortParts;
      return { [field]: direction.toLowerCase() };
    }
    return { [sort]: "asc" };
  }
  return { createdAt: "desc" };
}
```

**Step 4 ‚Äî Fix `getAdvertisements` helper (TSANY-04):**
Change signature from `(options: any, defaultFilters: any, ..., postProcessFilter?: (ads: any[]) => any[])` to:

```typescript
async function getAdvertisements(
  options: AdQueryOptions,
  defaultFilters: Record<string, unknown>,
  status: string,
  postProcessFilter?: (ads: unknown[]) => unknown[]
)

Inside the function body, fix the `parseInt` calls since `page` is now `string | number | undefined`:

const pageSize = options.pageSize ? parseInt(String(options.pageSize)) : 25;
```

The `orderBy: transformSortParameter(options.sort)` call needs a cast since `orderBy` expects a specific type:
```typescript

```

**Step 5 ‚Äî Fix all service method signatures (TSANY-01):**
Change every `options: any = {}` to `options: AdQueryOptions = {}`:

- `findOne(id: string | number, options: AdQueryOptions = {})`
- `findMany(options: AdQueryOptions = {})`
- `activeAds(options: AdQueryOptions = {})`
- `pendingAds(options: AdQueryOptions = {})`
- `archivedAds(options: AdQueryOptions = {})`
- `bannedAds(options: AdQueryOptions = {})`
- `rejectedAds(options: AdQueryOptions = {})`
- `abandonedAds(options: AdQueryOptions = {})`

For `findOne`, the spread `...options` into a strapi db query needs a cast:

For `findMany`, the spread `...options` also needs a cast:

**Step 4 ‚Äî Fix inline `options: any` locals in action methods (TSANY-06):**
In `actives`, `pendings`, `archiveds`, `banneds`, `rejecteds`, `abandoneds` ‚Äî each has the pattern:
```typescript
// BEFORE
const options: any = { ... };

// AFTER
const options: Record<string, unknown> = { ... };
```

Note: `options.pagination?.page` access may need adjustment since `pagination` would be `unknown`. Instead, type `options` with the shape it actually holds:
```typescript
const options: {
  page: number;
  pageSize: number;
  sort?: unknown;
  filters?: unknown;
  populate?: unknown;
} = { ... };
```

Or more simply ‚Äî since these are immediate construction objects, just remove the explicit type annotation and let TypeScript infer:
```typescript
// Remove the `: any` annotation entirely
const options = {
  ...query,
  page: query.pagination?.page ? parseInt(query.pagination.page as string, 10) : (query.page as number) || 1,
  pageSize: query.pagination?.pageSize ? parseInt(query.pagination.pageSize as string, 10) : (query.pageSize as number) || 25,
};
```
This avoids the `any` while letting TypeScript infer the actual shape.

**Step 5 ‚Äî Fix `filterClause: any` in `me()` (TSANY-06):**
```typescript
// BEFORE
const filterClause: any = { user: userId };

// AFTER
const filterClause: Record<string, unknown> = { user: userId };
```

The subsequent property assignments (`filterClause.active = true`, `filterClause.banned = false`, etc.) work fine with `Record<string, unknown>`.

**Step 6 ‚Äî Fix `ads.map((ad: any) =>)` in `me()` (TSANY-06):**
```typescript
// BEFORE
const adsWithStatus = ads.map((ad: any) => ({

// AFTER
const adsWithStatus = ads.map((ad) => ({
```
Remove the explicit `: any` annotation ‚Äî TypeScript infers the type from `entityService.findMany`.

**Step 7 ‚Äî Fix `filters as any` casts in `meCounts` (TSANY-07):**
Replace 5 occurrences of `filters: { ... } as any` with the Strapi SDK v5 cast pattern from AGENTS.md:
```typescript
// BEFORE
filters: { user: userId, active: true, ... } as any,

// AFTER

```

**Step 8 ‚Äî Fix `error.message` access in catch blocks:**
In `approveAd` and `bannedAd` controllers, `catch (error)` blocks access `error.message`. Since `catch (error: any)` is out of scope, check if any catch variable is currently typed. They are not (`catch (error)` without annotation) ‚Äî leave them as-is.

In `bannedAd` catch: `error.message.includes("permission")` ‚Äî this may throw a TypeScript error if `error` is `unknown`. Since the tsconfig has `strict: false` (and `noImplicitAny: false`), TypeScript does NOT enforce typed catch variables. Leave catch blocks untouched.
