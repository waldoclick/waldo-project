# Phase 49: Zoho + Facto + Other Services any Elimination — Plan 1

**Plan:** 49-01
**Goal:** Eliminate all `any` types from Zoho service/interfaces/http-client, Facto factory/config/SOAP services, Indicador service, Google types/sheets, Transbank types/service, and payment-gateway interface

**Wave:** 1 of 1
**Estimated tasks:** 3

## Context

Eleven files across six service directories contain all the `any` types for this phase:

1. `apps/strapi/src/services/zoho/zoho.service.ts` — 5 `Promise<any>` return types + 5 `{ data: any[] }` inline generics
2. `apps/strapi/src/services/zoho/interfaces.ts` — 4 `Promise<any>` return types in `IZohoService`
3. `apps/strapi/src/services/zoho/http-client.ts` — `params?: any`, `data: any` × 2 (post/put)
4. `apps/strapi/src/services/facto/factories/facto.factory.ts` — `datos: any`
5. `apps/strapi/src/services/facto/config/facto.config.ts` — `private client: any`, `getClient(): any`
6. `apps/strapi/src/services/facto/services/electronic-ticket.service.ts` — `(err: any, result: any)` SOAP callback
7. `apps/strapi/src/services/facto/services/electronic-invoice.service.ts` — `(err: any, result: any)` SOAP callback
8. `apps/strapi/src/services/indicador/indicador.service.ts` — `transformToEnglishFormat(data: any)`
9. `apps/strapi/src/services/google/types/google.types.ts` — `appendToSheet(data: any[])`
10. `apps/strapi/src/services/google/services/google-sheets.service.ts` — `appendToSheet(data: any[])`

13. `apps/strapi/src/services/payment-gateway/types/gateway.interface.ts` — `error?: any` × 2, `response?: any`

No new files. No new packages. All changes are in-place edits.

`tsc --noEmit` passes with zero errors before this phase. It must continue to pass after every task.

Catch block `error: unknown` narrowing is **out of scope** per REQUIREMENTS.md — do NOT change `catch (error)` parameter types.

#### electronic-ticket.service.ts and electronic-invoice.service.ts (TSANY-18)

Both files have identical `(err: any, result: any)` patterns — apply the same fix to each.

**Step 4 — Fix SOAP callback in `electronic-ticket.service.ts`:**

```typescript
// BEFORE
return new Promise((resolve, reject) => {
  client.emitirDocumento(params, (err: any, result: any) => {
    if (err) {
      console.error("Error en la llamada SOAP:", err);
      reject(err);
      return;
    }

    console.log("Respuesta de Facto:", JSON.stringify(result, null, 2));

    if (!result || !result.return || !result.return.resultado) {
      reject(new Error("Respuesta inválida del servicio"));
      return;
    }

    if (result.return.resultado.status !== "0") {
      reject(
        new Error(
          result.return.resultado.mensaje_error ||
            "Error al emitir documento"
        )
      );
      return;
    }

    resolve(result);
  });
});

// AFTER
return new Promise((resolve, reject) => {
  client.emitirDocumento(params, (err: unknown, result: unknown) => {
    if (err) {
      console.error("Error en la llamada SOAP:", err);
      reject(err instanceof Error ? err : new Error(String(err)));
      return;
    }

    console.log("Respuesta de Facto:", JSON.stringify(result, null, 2));

    const soapResult = result as {
      return?: {
        resultado?: { status?: string; mensaje_error?: string | null };
      };
    };

    if (!soapResult || !soapResult.return || !soapResult.return.resultado) {
      reject(new Error("Respuesta inválida del servicio"));
      return;
    }

    if (soapResult.return.resultado.status !== "0") {
      reject(
        new Error(
          soapResult.return.resultado.mensaje_error ||
            "Error al emitir documento"
        )
      );
      return;
    }

    resolve(result as IFactoLegacyResponse);
  });
});
```

Apply the exact same fix as Step 4 to `electronic-invoice.service.ts`. The callback body is identical — only the service class name and method name differ.
