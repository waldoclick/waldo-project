<objective>
Fix two bugs in `useAdAnalytics.ts` `purchase()` function: Strapi's `biginteger` fields serialize
to strings in JSON, so `order.amount` arrives as `"19990"` (string) — GA4 receives a string and
shows $0 revenue. Additionally, `item_id` falls back to empty string when `documentId` is absent,
when it should fall back to `transactionId` first.

Purpose: GA4 Ecommerce dashboard shows real revenue and real item IDs for every purchase event.
Output: Two-line fix in `useAdAnalytics.ts` + 2 new Vitest tests (14 total, all green).
</objective>

From apps/website/app/composables/useAdAnalytics.ts — purchase() function (lines 157–187):
```typescript
export interface PurchaseOrderData {
  documentId?: string;
  amount?: number;       // NOTE: may receive string from Strapi API despite number type
  totalAmount?: number;
  currency?: string;
  payment_response?: {
    buy_order?: string;
    authorization_code?: string;
  };
}

const purchase = (order: PurchaseOrderData) => {
  const transactionId =
    order.payment_response?.buy_order ?? order.documentId ?? "";
  const value = order.amount ?? order.totalAmount ?? 0;      // ← BUG: string passthrough
  const currency = order.currency ?? "CLP";

  const items: AnalyticsItem[] = [
    {
      item_id: order.documentId ?? "",                        // ← BUG: no transactionId fallback
      item_name: "Orden de pago",
      item_category: "Order",
      price: value,
      quantity: 1,
      currency,
    },
  ];

  pushEvent("purchase", [], {
    ecommerce: { transaction_id: transactionId, value, currency, items },
  }, "pack_purchase");
};
```

Root causes (from research):
- ECOM-01: Strapi `biginteger` → JSON string. `??` passes string through unchanged.
  Fix: `const value = Number(order.amount ?? order.totalAmount ?? 0);`
- ECOM-02: `item_id: order.documentId ?? ""` skips `transactionId` as fallback.
  Fix: `item_id: order.documentId || transactionId || ""`
