# Roadmap Archive: v1.24 Free Ad Submission

**Status:** ✅ SHIPPED 2026-03-09
**Phases:** 58-59
**Total Plans:** 2

## Overview

Introduced a dedicated `POST /api/payments/free-ad` endpoint in Strapi that handles the complete free ad submission lifecycle (credit validation, reservation linking, publishing, emails) without touching the existing `POST /api/payments/ad` or `ad.service.ts`. Wired `resumen.vue` to call the new endpoint via a two-step pattern: `save-draft` → `free-ad`. Also cleaned up dead code: `packCreate`/`packResponse` handlers removed from the payment controller, pack routes removed, result pages moved from `anunciar/` and `packs/` to `pagar/`.

## Milestone Summary

**Key Decisions:**

- `free-ad.service.ts` decoupled from `ad.service.ts` — zero shared imports, zero risk to existing paid flow
- Email failures non-fatal in free-ad flow — reservation + publication are critical, email is not
- Two-step free creation: `save-draft` then `free-ad` — mirrors the paid flow; `free-ad` endpoint requires an existing `ad_id`
- `freeAdCreate` permission must be granted to Authenticated role in Strapi admin panel after deploy
- `packCreate`/`packResponse` removed as dead code — unified payment flow uses `payments/ad` (ad+pack) only

**Issues Resolved:**

- Double-wrapped payload bug in `save-draft` and `payments/free-ad` calls (Strapi SDK wraps automatically)
- Route naming inconsistency: `ads/draft` → `ads/save-draft` corrected atomically

**Technical Debt Incurred:**

- None. `POST /api/payments/ad` and `ad.service.ts` left intact intentionally for backward compatibility.
