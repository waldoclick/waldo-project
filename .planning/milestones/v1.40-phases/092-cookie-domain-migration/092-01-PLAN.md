In BOTH composables, add the old-cookie cleanup line **immediately before** `await strapiLogout()`. This clears any pre-migration host-only `waldo_jwt` cookie (no domain attribute) that may be lingering in the browser after the domain migration goes live.

The cleanup is client-only (`document.cookie`) — wrap with `if (import.meta.client)` to be safe on SSR.

**apps/website/app/composables/useLogout.ts** — add before `await strapiLogout()`:
```typescript
// Clear the old host-only cookie (pre-migration, no domain attr) to prevent zombie sessions
if (import.meta.client) {
  document.cookie = "waldo_jwt=; path=/; max-age=0";
}
```

**apps/dashboard/app/composables/useLogout.ts** — add before `await strapiLogout()`:
```typescript
// Clear the old host-only cookie (pre-migration, no domain attr) to prevent zombie sessions
if (import.meta.client) {
  document.cookie = "waldo_jwt=; path=/; max-age=0";
}
```

The `domain` attribute is intentionally OMITTED from the cleanup line — `document.cookie = "waldo_jwt=; path=/; max-age=0"` targets the host-only cookie (no domain attr). The new shared cookie (with domain attr) is cleared by `strapiLogout()` using the useCookie() instance that knows the domain.

Do not modify any other part of either composable.
