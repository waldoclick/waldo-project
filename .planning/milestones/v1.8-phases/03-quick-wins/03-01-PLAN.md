<objective>
Eliminate the double API call on mount in all six Ads list components, and give each ads section its own isolated pagination/filter state in the settings store.

Purpose: Every ads list view currently makes two fetches on mount (one from watch with immediate:true, one from onMounted). This is wasteful and can cause race conditions. Additionally, all six components share the single "ads" section key in the settings store, meaning changing the page in Pending Ads pollutes the page number for Active Ads and all others.

Output: Six corrected component files (no onMounted fetch) and an expanded settings store (six distinct ads section keys replacing the single "ads" key).
</objective>
