### Task 1.1 — BUG-01: Add type declaration to `microdata.ts` plugin

**File:** `apps/website/app/plugins/microdata.ts`
**Requirement:** BUG-01

**Why this is needed:**
The plugin provides `$setStructuredData` via Nuxt's `provide` mechanism but has no `declare module "#app"` type augmentation. Pages compensate with `as unknown as { $setStructuredData }` casts. The `seo.ts` plugin (same file, same project) already demonstrates the correct pattern.

**Action — append type declaration to `microdata.ts`:**

Current file ends at line 23. Add the following block after the closing `});`:

```typescript
declare module "#app" {
  interface NuxtApp {
    $setStructuredData: (data: object) => void;
  }
}
```

**Full resulting file:**
```typescript
// plugins/structured-data.ts
import { useHead } from "#app";

export default defineNuxtPlugin(() => {
  // Función para agregar datos estructurados
  const setStructuredData = (data: object) => {
    useHead({
      script: [
        {
          type: "application/ld+json",
          children: JSON.stringify(data),
        },
      ],
    });
  };

  // Exponer la función globalmente
  return {
    provide: {
      setStructuredData,
    },
  };
});

declare module "#app" {
  interface NuxtApp {
    $setStructuredData: (data: object) => void;
  }
}
```

**Verification:**
- TypeScript no longer requires `as unknown as` cast on pages that call `$setStructuredData`
- Inspecting any page's source that calls `$setStructuredData` shows `<script type="application/ld+json">` in the rendered HTML

```typescript

```

**Action 3 — `anuncios/[slug].vue` line 48–49:**

### Task 1.3 — BUG-03: Fix `console.client.ts` — restore diagnostic console methods

**File:** `apps/website/app/plugins/console.client.ts`
**Requirement:** BUG-03

**Why this is needed:**
The plugin currently silences `console.warn`, `console.error`, and `console.info` in production. This hides operational errors from developers inspecting the live site. Only `console.log` and `console.debug` (noisy development output) should be suppressed.

**Current file (6 lines of logic):**
```typescript
export default defineNuxtPlugin(() => {
  if (import.meta.dev) {
    // console.log("🔧 Console plugin initialized in development mode");
  } else {
    // console.log = () => {};
    console.debug = () => {};
    console.warn = () => {};    // ← BUG: suppresses warnings
    console.error = () => {};   // ← BUG: suppresses errors
    console.info = () => {};    // ← BUG: suppresses info
  }
});
```

**Full resulting file:**
```typescript
export default defineNuxtPlugin(() => {
  if (!import.meta.dev) {
    console.log = () => {};
    console.debug = () => {};
    // console.warn, console.error, console.info intentionally NOT suppressed
  }
});
```

**Verification:**
- In production (or with `NODE_ENV=production` locally), open DevTools
- `console.error("test")` in a component appears in DevTools
- `console.warn("test")` in a component appears in DevTools
- `console.log("test")` does NOT appear
- `console.debug("test")` does NOT appear

### Task 1.4 — BUG-04: Add `await` + key to `mis-anuncios.vue` `useAsyncData`

**File:** `apps/website/app/pages/cuenta/mis-anuncios.vue` (line 80–87)
**Requirement:** BUG-04

**Why this is needed:**
Without `await`, Nuxt's SSR pass completes before the data fetch resolves — the page renders empty on the server and then re-fetches on the client, causing a hydration mismatch. Without a key, Nuxt cannot serialize the payload for SSR→CSR hand-off.

**Current code (lines 80–87):**
```typescript
useAsyncData(async () => {
  const counts = await userStore.loadUserAdCounts();
  // Populate all tab counts at once from the single response
  for (const tab of tabs.value) {
    tab.count = counts[tab.value] ?? 0;
  }
  await loadAds(); // Cargar los anuncios del filtro actual
});
```

**Replacement:**
```typescript
await useAsyncData("mis-anuncios", async () => {
  const counts = await userStore.loadUserAdCounts();
  // Populate all tab counts at once from the single response
  for (const tab of tabs.value) {
    tab.count = counts[tab.value] ?? 0;
  }
  await loadAds(); // Cargar los anuncios del filtro actual
});
```

**Verification:**
- Load `/cuenta/mis-anuncios` directly via URL (SSR) — ad counts and ad list render immediately
- Navigate to the page via client-side link — same data renders without a flash
- Both views show the same tab counts
