# Requirements: Waldo Project — v1.22 Checkout Flow UI

**Defined:** 2026-03-08
**Core Value:** Los usuarios pueden publicar y gestionar avisos de forma confiable, con pagos que funcionan sin fricción — independientemente de la pasarela utilizada.

## v1.22 Requirements

### Página de Pago (`/pagar`)

- **PAGE-01**: Existe la página `/pagar/index.vue` con `middleware: "auth"` — usuarios no autenticados son redirigidos al login
- **PAGE-02**: La página tiene `useSeoMeta({ robots: "noindex, nofollow" })` — no indexable por motores de búsqueda
- **PAGE-03**: La página renderiza `CheckoutDefault` como único contenido tras el header

### Componente PaymentAd

- **PAD-01**: Existe `PaymentAd.vue` siguiendo BEM `payment payment--ad`
- **PAD-02**: `PaymentAd` muestra la primera imagen del gallery del ad (placeholder si no hay imagen)
- **PAD-03**: `PaymentAd` muestra el nombre del aviso (`adStore.ad.name`)
- **PAD-04**: `PaymentAd` muestra el precio del aviso formateado en CLP si existe
- **PAD-05**: `PaymentAd` incluye un botón "Editar" via `ButtonEdit` que lleva a `/anunciar`

### Componente PaymentGateway

- **PGW-01**: Existe `PaymentGateway.vue` siguiendo BEM `payment payment--gateway`
- **PGW-02**: `PaymentGateway` muestra un checkbox con la opción "WebPay" marcada y deshabilitada (decorativo — única pasarela disponible)
- **PGW-03**: `PaymentGateway` está preparado para agregar pasarelas adicionales en el futuro sin modificar la estructura

### Formulario de Checkout (`FormCheckout`)

- **FORM-01**: `FormCheckout` tiene `<script setup lang="ts">` con tipos correctos (no `any` implícito)
- **FORM-02**: El texto intro del formulario refleja el contexto de pago ("Confirma tu pago")
- **FORM-03**: Cada sección del formulario tiene un `<h3 class="form__section-title">` descriptivo
- **FORM-04**: El orden de secciones es: Tu anuncio → Tipo de publicación → Destacado → Boleta o factura → Pasarela de pago
- **FORM-05**: El botón de acción principal dice "Ir a pagar" (no "Continuar")
- **FORM-06**: Dead code eliminado: `handleformBack`, evento `formBack`, `@back` binding

### Redirección desde Flujos Existentes

- **REDIR-01**: El flujo de creación de anuncio (`/anunciar/resumen`) redirige a `/pagar` cuando `hasToPay === true` en lugar de iniciar el pago directamente
- **REDIR-02**: El flujo de compra de packs redirige a `/pagar` en lugar de iniciar el pago directamente
- **REDIR-03**: La lógica de pago (llamada a `payments/ad`, WebPay redirect, manejo de errores) se mueve de `resumen.vue` a `CheckoutDefault.vue` / `FormCheckout.vue`

### Estilos

- **SCSS-01**: `_payment.scss` tiene bloque `&--ad` con estilos para la tarjeta preview del aviso
- **SCSS-02**: `_payment.scss` tiene bloque `&--gateway` con estilos para la selección de pasarela
- **SCSS-03**: `_form.scss` tiene `&__section-title` para los títulos de sección dentro del checkout

## Out of Scope

| Feature | Reason |
|---------|--------|
| Conexión real del botón "Ir a pagar" con el backend | Milestone posterior |
| Compra de packs desde `/pagar` (múltiples ads) | Milestone posterior |
| Múltiples pasarelas de pago | Milestone posterior — solo WebPay hoy |
| TypeScript fixes en PaymentMethod/PaymentFeatured/PaymentInvoice | Deuda técnica — milestone dedicado |
