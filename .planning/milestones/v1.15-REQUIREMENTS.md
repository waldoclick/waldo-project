# Requirements — v1.15 Website SEO Audit

## Milestone Goal

Complete a full SEO audit of `apps/website`: fix the `$setSEO` plugin to emit the full Open Graph + Twitter Card tag set; replace all hardcoded production URLs with `config.public.baseUrl`; add missing SEO coverage to 3 pages; restore user profile page SEO; add home page structured data; fix JSON-LD accumulation on SPA navigation; add `noindex` to private pages; and verify the sitemap is accurate. `nuxt typecheck` must pass with zero errors.

## Requirements

### SEO-01: Extend `$setSEO` plugin to emit full OG + Twitter Card tags

**What:** Update `apps/website/app/plugins/seo.ts` to call `useSeoMeta` with the full tag set: `title`, `ogTitle`, `description`, `ogDescription`, `ogImage`, `ogUrl`, `ogType` (defaults to `"website"`), `twitterCard` (defaults to `"summary_large_image"`), `twitterTitle`, `twitterDescription`.

**Why:** The current plugin only sets `title`, `description`, and `ogImage`. When the page is shared on Twitter/X or iMessage the link preview has no title — `ogTitle` is the field that social crawlers read, not the plain `<title>` tag.

**Acceptance criteria:**
- `seo.ts` passes all new params to `useSeoMeta` (ogTitle mirrors title if not separately provided, ogUrl uses the `url` param if given)
- No existing `$setSEO` call signatures need to change (all new params are either derived from existing ones or have safe defaults)
- TypeScript interface for `$setSEO` params updated to include optional `ogType` and `twitterCard` overrides

**Phase:** 35
**Plan:** 35-01

### SEO-02: Replace hardcoded `https://waldo.click` URLs with `config.public.baseUrl`

**What:** In every page file under `apps/website/app/pages/` that passes a hardcoded `https://waldo.click/...` string to `$setSEO` or `$setStructuredData`, replace the string with a template expression using `const config = useRuntimeConfig()` and `config.public.baseUrl`.

**Why:** The site config already has `baseUrl` in `runtimeConfig.public` for exactly this purpose. Hardcoded production URLs mean staging environments and local dev generate incorrect canonical/OG URLs, and any domain change would require touching every page file.

**Acceptance criteria:**
- `grep -r "https://waldo.click" apps/website/app/pages/` returns zero results (excluding comments and commented-out blocks)
- `config.public.baseUrl` is used for all absolute URL construction in SEO calls
- Where `config` is already imported on the page, no duplicate `useRuntimeConfig()` call is added
- `nuxt typecheck` passes

**Phase:** 35
**Plan:** 35-01

### SEO-03: Add missing SEO to `packs/index.vue`, `cuenta/mis-ordenes.vue`, `cuenta/mis-anuncios.vue`

**Why:** Pages without SEO meta tags display generic browser-tab titles and produce empty social previews. `mis-ordenes` and `mis-anuncios` also need `noindex` since they are behind auth — but a title is still required for correct browser tab labeling.

**Acceptance criteria:**
- `packs/index.vue` has `$setSEO({ title, description, imageUrl, url })` and `$setStructuredData` with `WebPage` type
- `cuenta/mis-ordenes.vue` has `$setSEO` with descriptive title + `noindex`
- `cuenta/mis-anuncios.vue` has `$setSEO` with descriptive title + `noindex`

**Phase:** 35
**Plan:** 35-02

### SEO-05: Add WebSite + Organization structured data to home page

**What:** Add a `$setStructuredData` call to `apps/website/app/pages/index.vue` that emits a `WebSite` schema (with `SearchAction` sitelinks search box) and an `Organization` schema.

**Why:** The home page is the highest-authority page. Google uses `WebSite` schema to enable the sitelinks search box in SERPs and `Organization` to display the Knowledge Panel. The home page currently has no JSON-LD at all.

**Acceptance criteria:**
- `index.vue` calls `$setStructuredData([WebSite, Organization])` after the `$setSEO` call
- `WebSite` includes `url`, `name`, `description`, and a `SearchAction` pointing to `/anuncios?s={search_term_string}`
- `Organization` includes `url`, `name`, `logo`, `sameAs` (empty array acceptable for now)
- `nuxt typecheck` passes

**Phase:** 35
**Plan:** 35-02

### SEO-06: Fix JSON-LD accumulation in `microdata.ts` plugin

**What:** Change `apps/website/app/plugins/microdata.ts` so that each call to `$setStructuredData` replaces the existing JSON-LD `<script>` tag rather than appending a new one. Use a stable `useHead` key (e.g. `{ key: 'structured-data' }` on the script entry).

**Why:** On SPA navigation, each route change calls `$setStructuredData` which appends a new `<script type="application/ld+json">` tag to `<head>`. By the time a user has navigated 3 pages, the head contains 3 stale JSON-LD blocks. Google's structured data testing tool and crawlers can misread multiple conflicting blocks.

**Acceptance criteria:**
- After navigating home → ad detail → back home, the `<head>` contains exactly one `<script type="application/ld+json">` tag (not two or three)
- The `key` option on the script entry ensures Nuxt's `useHead` deduplicates on re-render
- Existing `$setStructuredData` call signatures are unchanged

**Phase:** 35
**Plan:** 35-03

### SEO-07: Add `noindex, nofollow` to private and transactional pages

**What:** Add `useSeoMeta({ robots: 'noindex, nofollow' })` to all pages that are behind auth or have no indexing value: `cuenta/index.vue`, `cuenta/perfil/index.vue`, `cuenta/perfil/editar.vue`, `cuenta/avatar.vue`, `cuenta/cover.vue`, `cuenta/username.vue`, `cuenta/cambiar-contrasena.vue`, `anunciar/index.vue`, `anunciar/resumen.vue`, `anunciar/gracias.vue`, `anunciar/error.vue`, `packs/comprar.vue`, `packs/gracias.vue`, `packs/error.vue`, `login/index.vue`, `registro.vue`, `recuperar-contrasena.vue`, `restablecer-contrasena.vue`.

**Why:** `robots.txt` already disallows these paths, but defense-in-depth with `noindex` in the page HTML is best practice — it ensures the directive is honoured even if robots.txt is misconfigured or a page is linked to directly.

**Acceptance criteria:**
- All listed pages emit `<meta name="robots" content="noindex, nofollow">` in their `<head>`
- The directive is added via `useSeoMeta({ robots: 'noindex, nofollow' })` alongside existing `$setSEO` calls, or as the sole SEO call where no `$setSEO` exists
- Public pages (`/anuncios`, `/anuncios/[slug]`, `/preguntas-frecuentes`, `/politicas-de-privacidad`, `/contacto`, `/packs/index`, `/[slug]`, home) are NOT given noindex

**Phase:** 35
**Plan:** 35-02

### SEO-08: Audit and improve sitemap configuration

**What:** In `apps/website/nuxt.config.ts`, restructure the static `sources` entries into `urls` entries with `loc`, `changefreq`, and `priority` fields. Verify the dynamic ad URL generator correctly uses populated `slug` and `updatedAt` fields. Confirm `exclude` list covers all non-indexable paths.

**Why:** The current sitemap `sources` array only lists paths without metadata. Sitemap entries without `changefreq` and `priority` are processed with lower confidence by crawlers. The static entries also currently lack these hints while the dynamic ad entries already have them — inconsistency.

**Acceptance criteria:**
- Each static page entry has `loc`, `changefreq`, and `priority` values appropriate to the page type
- The dynamic ad URL function is verified working (no changes required if correct)
- `exclude` list is verified to include all auth/transactional path patterns
- Sitemap renders correctly when accessed at `/sitemap.xml`

**Phase:** 35
**Plan:** 35-03
