<objective>
Extend useAdAnalytics composable with a `purchase()` method that builds event items from raw order data, and add optional `flow` discriminator support to `pushEvent()`.

Purpose: The purchase event cannot rely on ad store analytics state because `adStore.clearAll()` is called in `onMounted` on /pagar/gracias. The composable must accept order data directly to build the GA4 ecommerce `purchase` payload.

Output: Updated `useAdAnalytics.ts` with `purchase(order)` method and `flow` param support in `pushEvent()`.
</objective>

export const useAdAnalytics = () => {

Current `pushEvent` hardcodes `flow: "ad_creation"`. It needs to accept an optional `flow` param so callers can pass `"pack_purchase"`.

The `purchase()` method must build items from order data, NOT from `adStore.getAnalytics`, because the store is cleared before this can fire. The order object available in gracias.vue has this shape:

```typescript
interface OrderData {
  documentId?: string;
  amount?: number;
  totalAmount?: number;
  currency?: string;
  payment_response?: {
    buy_order?: string;
    authorization_code?: string;
  };
}
```

Items for the purchase event: one item representing the pack/order, built from order fields.
