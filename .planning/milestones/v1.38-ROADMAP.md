## Milestone Summary

**Key Decisions:**

- `Number()` coercion for Strapi biginteger fields — always wrap numeric fields from API responses before passing to GA4
- `||` (not `??`) for item_id fallback chains — empty string must trigger fallback
- `watch(adData, { immediate: true })` + `purchaseFired` ref guard for SSR-safe analytics firing — mirrors `pagar/gracias.vue` pattern
- Free-ad purchase event uses `amount: 0` (not undefined) — enables GA4 funnel comparison between free and paid conversions
- `viewItemFired` boolean ref + slug-change reset watcher — Nuxt reuses `[slug].vue` component across navigations
- `resolveSearchTerm()` maps commune ID → name via filterStore for human-readable `search_term`
- All engagement/lifecycle/content GA4 events use empty items array `[]` — not ecommerce events
- `signUp()` has no method param — always pushes `method: 'email'`; Google sign-up uses OAuth flow
- `ga4-engagement-events` GTM tag uses `{{Event}}` as event name — dynamic pickup from dataLayer, no hardcoding
- `@click.capture` in `AdSingle.vue` for contactSeller — fires regardless of internal copy button propagation

**GTM Setup Completed:**

- Trigger `engagement-events`: Custom Event, regex `contact|generate_lead|sign_up|login|article_view`
- Tag `ga4-engagement-events`: GA4 Event, `{{Event}}` as event name, triggered by `engagement-events`
- Published as GTM Version 6

**Flow strings established:**
- `user_engagement` — contactSeller, generateLead
- `user_lifecycle` — signUp, login
- `content_engagement` — articleView

**Issues Resolved:**

- ECOM-01: GA4 purchase revenue was $0 due to Strapi biginteger fields serializing as strings in JSON — fixed with `Number()` coercion
- ECOM-02: GA4 item_id was empty string — fixed with `||` fallback chain (not `??`)
- ECOM-03: Free ad creation conversions were invisible in GA4 — fixed with `watch(adData)` in `/anunciar/gracias.vue`
