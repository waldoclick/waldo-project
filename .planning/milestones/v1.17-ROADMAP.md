## Key Decisions

- `strapi.db.query` used over Strapi service to bypass content-API sanitizer that strips `filters[role]` for regular JWTs — non-forgeable server-side role enforcement
- Inline sanitize pattern (spread + omit sensitive fields) replaces `getDetailedUserData` to eliminate N+1 on users list endpoint
- `dsn: undefined` pattern (not conditional init) used in Sentry config files — SDK-supported approach, skips all instrumentation with zero overhead
- `enabled: process.env.NODE_ENV === 'production'` in Strapi Sentry plugin unloads the plugin entirely in dev/staging

## Milestone Summary

**Shipped:** Server-enforced role filter for dashboard users table (non-forgeable, with TDD), and production-only Sentry guard across all three apps.
