## Milestone Summary

**Key Decisions:**

- `$setSEO` extended to emit full OG + Twitter tag set — `ogTitle`/`ogDescription`/`twitterTitle`/`twitterDescription` derived from existing `title`/`description` params — zero call-site changes required
- `twitterCard` cast to union type literal in `useSeoMeta` — needed because `useSeoMeta` expects a specific union, not plain `string`
- `config.public.baseUrl` used for all absolute URL construction — single source of truth, environment-agnostic
- `useHead` key on JSON-LD script entry (`key: "structured-data"`) prevents accumulation — Nuxt merges `useHead` calls with matching keys natively
- `noindex` via `useSeoMeta` as defense-in-depth — `robots.txt` already disallows private paths; inline `noindex` survives misconfiguration
- `nuxt-simple-sitemap` cannot have both static array and async function under same `urls` key — merged into single async function

**Issues Resolved:**

- Social sharing previews (Twitter/X, iMessage, WhatsApp, Slack) had no title — `ogTitle` was missing from `$setSEO` plugin
- SPA navigation accumulated multiple JSON-LD `<script>` tags — fixed by `useHead` key deduplication
- Hardcoded production URLs broke SEO on staging/local — all replaced with `config.public.baseUrl`
- User profile page `[slug].vue` had SEO disabled since v1.9 — restored with working `watch()` pattern
