<task type="auto">
  <name>Task 2: Fix meStore cache invalidation in FormProfile after profile save</name>
  <files>apps/website/app/components/FormProfile.vue</files>
  <action>
    **Problem:** After FormProfile saves the profile and calls `fetchUser()`, `meStore.me` still holds the old (incomplete) profile data. On the next navigation, `isProfileComplete()` returns the cached stale result, causing the guard to redirect back to `/onboarding` in a loop.

    **Fix:** In `FormProfile.vue`, after the `await fetchUser()` call (around line 680), add `useMeStore().reset()` to invalidate the cached `me` value. This forces `isProfileComplete()` to re-fetch on the next call, seeing the now-complete profile.

    The change is exactly:
    ```typescript
    await fetchUser();
    useMeStore().reset();
    ```

    `useMeStore` is auto-imported by Nuxt (Pinia store in stores/ directory). No new import needed.
