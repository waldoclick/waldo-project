# Milestone v1.30: Blog Public Views

**Status:** ✅ SHIPPED 2026-03-13
**Phases:** 065–068
**Total Plans:** 8

## Overview

Exposed the Article content type as a public blog on the website. Built a full listing page (`/blog`) with category filtering, sort order, and pagination, plus an individual article detail page (`/blog/[slug]`) with rendered Markdown body, cover gallery, sidebar, and related articles. All blog components replicated from ads equivalents with blog-specific BEM namespaces. Full SEO and structured data on both pages.

## Phases

### Phase 065: Strapi Slug Field

**Goal**: The Article content type exposes a unique `slug` field auto-generated from the title, and the public API returns it alongside all related fields
**Depends on**: Nothing (v1.29 Article content type exists)
**Requirements**: BLOG-01, BLOG-02
**Plans**: 1 plan

Plans:
- [x] 065-01: Add slug uid field to Article schema + beforeCreate/beforeUpdate lifecycle hooks with Jest tests

**Details:**
- `slug` field added to Article Strapi schema as `uid` type targeting `title`
- `beforeCreate` / `beforeUpdate` lifecycle hooks auto-generate slug via `slugify` with `strict: true` (strips accents, special chars — `¡Artículo!` → `articulo`)
- `beforeUpdate` fetches existing title via `entityService.findOne` before overwriting to prevent unnecessary regeneration
- `GET /api/articles` returns articles with `slug`, `categories`, `cover`, `gallery` populated
- 6 Jest tests covering slug generation, update, uniqueness

## Milestone Summary

**Key Decisions:**
- Blog components replicated from ads equivalents with blog-specific BEM namespaces — not reused directly (user requirement)
- `slug` as `uid` type (not `string`) — gives admin auto-generation UI + uniqueness enforcement
- `article.gallery` (GalleryItem[] with `.url`) used for GalleryDefault — not `article.cover` (Media[], no direct `.url`)
- `useArticlesStore` has no persist — article list is volatile (changes with filters/pagination)
- `@type: "Blog"` for listing page structured data; `@type: "BlogPosting"` for detail page
- Related articles: same-category first, fill with most-recent fallback, deduplicate, slice to 6
- `HeroArticles` accepts zero props — blog index hero is always static
- `showError` uses `statusMessage` (not `description`) — correct H3Error field in NuxtError
- `slugify` with `strict: true` for clean URL slugs

**Issues Resolved During Milestone:**
- Duplicate "Inicio" breadcrumb in blog listing hero (removed)
- `FilterArticles` BEM class mismatch (`selector__select` → `select`) corrected after verification
- Cover images null-guarded (API may return null `cover`/`gallery`/`categories`)
- Markdown body not rendering (fixed via `marked` parse in `blog/[slug].vue`)
- Article body paragraph spacing (added `p + p` margin in SCSS)
- Article body being reset to null on dashboard save (missing field in edit.vue interface)
- Article update using numeric id causing 400 (switched to `documentId`)
- Dashboard article edit/preview pages showing blank (body field wiring fixed)

**Issues Deferred:**

- Reading time estimate (BLOG-F02) — future milestone
- Author attribution (BLOG-F03) — requires author entity in Strapi
