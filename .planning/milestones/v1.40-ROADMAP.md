# Milestone v1.40: Shared Authentication Session

**Status:** ✅ SHIPPED 2026-03-16
**Phases:** 091–092
**Total Plans:** 3

## Overview

Enables login-once access across both Waldo subdomains (`waldo.click` and `dashboard.waldo.click`) by sharing the JWT cookie at the `.waldo.click` domain level. Also centralizes all dashboard logout logic into a single composable and eliminates zombie sessions from pre-migration host-only cookies.

## Phases

### Phase 091: Dashboard useLogout Composable

**Goal**: Dashboard logout is centralized in a single composable — every call site uses it, and the old-cookie cleanup can be applied in one place
**Depends on**: Nothing (first phase of milestone)
**Requirements**: SAFE-01
**Plans**: 1 plan

Plans:

- [x] 091-01: Create useLogout composable + meStore.reset() + migrate 3 call sites

**Details:**

Created `apps/dashboard/app/composables/useLogout.ts` mirroring the website pattern: resets appStore, meStore, searchStore, then calls `strapiLogout()` and navigates to `/auth/login`. Added `reset()` function to `meStore`. Migrated all 3 logout call sites (DropdownUser.vue, FormVerifyCode.vue, guard.global.ts). `nuxt typecheck` exits 0.

### Phase 092: Cookie Domain Migration

**Goal**: Users authenticated on one subdomain are automatically recognized on the other — login once, access both apps; logout anywhere clears both
**Depends on**: Phase 091
**Requirements**: SESS-01, SESS-02, SESS-03, SESS-04, SESS-05, SESS-06, SAFE-02, SAFE-03
**Plans**: 2 plans

Plans:

- [x] 092-01: Add COOKIE_DOMAIN conditional to both nuxt.config.ts + old-cookie cleanup in both useLogout composables
- [x] 092-02: Document COOKIE_DOMAIN in both .env.example files + human-verify checkpoint

**Details:**

Both apps now emit `Set-Cookie: waldo_jwt=...; Domain=.waldo.click` when `COOKIE_DOMAIN=.waldo.click` is set in production. When unset (local dev), behavior is unchanged — host-only cookies issued as before. Both useLogout composables explicitly clear the pre-migration host-only `waldo_jwt` cookie before calling `strapiLogout()`. COOKIE_DOMAIN documented as commented-out examples in both `.env.example` files with production/staging values.

## Milestone Summary

**Key Decisions:**

- Conditional spread pattern `...(COOKIE_DOMAIN ? { domain: COOKIE_DOMAIN } : {})` keeps the cookie block clean and TypeScript-safe
- `import.meta.client` guard for `document.cookie` — ensures SSR safety without runtime errors in Nitro
- No domain attr on old-cookie cleanup line — targets only the pre-migration host-only cookie; `strapiLogout()` clears the new shared cookie
- `import { useStrapiAuth, navigateTo } from '#imports'` in composable — required for Nuxt auto-import interception

**Issues Resolved:**

- Zombie sessions from pre-migration host-only `waldo_jwt` cookies — resolved by explicit cleanup in both logout composables
- Scattered dashboard logout logic across 3 call sites — centralized in single `useLogout.ts` composable

**Issues Deferred:**

- POLL-01: Post-logout website Pinia stores reset when logout originates from dashboard (minor stale-data UX; stores are localStorage-scoped, no security risk)
- Formal staging cross-domain verification (SESS-01–06 code is complete; env var must be set in staging to execute)

**Technical Debt Incurred:**

- None — purely additive changes with zero breaking behavior
