# Coding Conventions

**Analysis Date:** 2026-03-10

## Naming Patterns

**Files:**
- Kebab case (`kebab-case`) for Vue pages and components inside the `pages/` and `components/` directories.
- Pascal case (`PascalCase`) used for TypeScript files involving utilities, stores, or composables.
- Service-prefixed names used under the `services/` directory in `apps/strapi` (e.g., `user.service.ts`).

**Functions:**
- `camelCase` pattern for regular functions and methods (e.g., `loadUserData`).
- Event-handlers suffix standard: functions align with the context in composable files (e.g., `onInputValidate()`).

**Variables:**
- Variables strictly in `camelCase`. Avoid abbreviating until universally recognized terms (e.g., `authToken` vs cutting redundance anonym).
