Transformation rule for each site:

- `<DATA>` = the EXISTING metadata object, minus the field that became `actor`. Example: a site that today logs `{ userId, adId }` becomes `actor: userId, actor_type: "plugin::users-permissions.user", data: { adId }`. Every other field that is logged today stays under `data` — drop NOTHING. Webpay references (buyOrder, token_ws, TBK_TOKEN, tbkToken, sessionId) that are currently logged stay under `data` unchanged.
