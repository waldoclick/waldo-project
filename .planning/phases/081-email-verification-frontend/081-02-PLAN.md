<objective>
Add an inline "resend confirmation" section to both `FormLogin.vue` components (website + dashboard) that appears when login fails because the account email is unconfirmed.

Purpose: REGV-05 requires an actionable error — not just a Swal explaining the problem. Currently both apps show a generic Swal message. The fix shows an inline section within the form with the user's email address and a clickable "Reenviar confirmación" button.

Output:
- Both `FormLogin.vue` files detect `"Your account email is not confirmed"` and show inline resend UI instead of Swal
- Resend calls `POST /api/auth/send-email-confirmation` (native Strapi endpoint)
- Unit tests covering the REGV-05 behavior
</objective>

<!-- New reactive state to add to script: -->
```typescript
const showResendSection = ref(false);
const unconfirmedEmail = ref('');
const resending = ref(false);

const handleResendConfirmation = async () => {
  resending.value = true;
  try {
    await client('/auth/send-email-confirmation', {
      method: 'POST',
      body: { email: unconfirmedEmail.value },
    });
    Swal.fire(
      "Correo enviado",
      "Hemos enviado un nuevo enlace de confirmación a tu correo.",
      "success"
    );
    showResendSection.value = false;
  } catch {
    Swal.fire("Error", "No se pudo enviar el correo. Inténtalo de nuevo.", "error");
  } finally {
    resending.value = false;
  }
};
```
