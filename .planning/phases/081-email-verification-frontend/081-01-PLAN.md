<objective>
Fix `FormRegister.vue` to prevent auth state corruption when email confirmation is active, then create the `/registro/confirmar` page that users land on after registering.

Purpose: Phase 082 will activate Strapi's `email_confirmation` toggle. When active, `POST /api/auth/local/register` returns `{ user }` with no JWT. The current code calls `useStrapiAuth().register()` which internally calls `setToken(response.jwt)` — passing `undefined` corrupts the auth store. This plan fixes the registration flow before that toggle is active, and builds the confirmation page users will see.

Output:
- `FormRegister.vue` using `useStrapiClient()` directly with `if (response.jwt)` guard
- `/registro/confirmar.vue` new page with email + resend button + 60s cooldown
- Unit tests covering the no-JWT redirect behavior
</objective>

```vue
const resendCooldown = ref(60);
let timer = null;
const startCountdown = () => {
  resendCooldown.value = 60;
  timer = setInterval(() => {
    resendCooldown.value--;
    if (resendCooldown.value <= 0) { clearInterval(timer); timer = null; }
  }, 1000);
};
onMounted(() => {
  if (!registrationEmail.value) { router.replace('/registro'); return; }
  startCountdown();
});
onUnmounted(() => { if (timer) clearInterval(timer); });
```
