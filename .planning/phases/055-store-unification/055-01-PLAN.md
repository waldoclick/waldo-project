# Plan 055-01: Create usePacksList composable and migrate components and composables

## Goal

Create a `usePacksList` composable that fetches packs directly from Strapi (with a TTL cache guard). Then migrate `PaymentMethod.vue`, `PackMethod.vue`, `useAdPaymentSummary.ts`, and `usePackPaymentSummary.ts` to use it instead of `usePacksStore()`.

## Context

`PaymentMethod.vue` is used in `FormCheckout.vue` and `FormCreateOne.vue`. `PackMethod.vue` is used in `FormPack.vue`. `useAdPaymentSummary` is called in 6 components and 1 page. Adding `packs` as a prop/parameter to all of those call sites would require extensive threading â€” instead, a shared `usePacksList` composable that manages its own Strapi fetch and cache is the clean solution. This composable has no Pinia dependency.

## Tasks

<task id="1" title="Create usePacksList composable">

**File:** `apps/website/app/composables/usePacksList.ts`

Create a new composable that loads packs from Strapi and returns them as a reactive ref. Use a module-level cache (packs ref + lastFetch number) so multiple call sites within the same page share one fetch, not multiple. Apply the same 30-minute TTL guard from `packs.store.ts` (1 800 000 ms).

```typescript
import type { Pack } from "@/types/pack";

const _packs = ref<Pack[]>([]);
let _lastFetch = 0;
const TTL = 1_800_000;

export const usePacksList = () => {
  const strapi = useStrapi();

  const loadPacks = async () => {
    const now = Date.now();
    if (_packs.value.length > 0 && now - _lastFetch < TTL) return;
    const response = await strapi.find("ad-packs", { populate: "*" });
    _packs.value = response.data as unknown as Pack[];
    _lastFetch = Date.now();
  };

  return {
    packs: readonly(_packs),
    loadPacks,
  };
};
```

Use `readonly()` on the returned ref to prevent accidental mutation. Import `readonly` from `vue` â€” do NOT import `ref` from `vue` explicitly (Nuxt auto-imports it), but `readonly` must be imported explicitly since it is not always auto-imported. Confirm by checking `apps/website/.nuxt/imports.d.ts` or trusting Nuxt auto-import â€” if uncertain, add `import { ref, readonly } from "vue"` at the top for safety.

**Verification:** File exists at the correct path with exported `usePacksList`.
</task>

<task id="2" title="Migrate PaymentMethod.vue to use usePacksList">

The component already has an `onMounted` (for UI init); add pack loading there since the comment says "packs pre-loaded by parent page" â€” but in the new architecture, the component loads its own packs via `usePacksList` (the TTL cache ensures no double-fetching if the parent page also loaded them via `useAsyncData`).

2. Remove: `const packsStore = usePacksStore();`

The `<script setup>` tag lacks `lang="ts"` â€” add `lang="ts"` and add the `PackType` import already present. Remove the unused `computed` import if `packs` computed is removed.

Final `<script setup lang="ts">` imports:
```typescript

import { useAdStore } from "@/stores/ad.store";
import type { PackType } from "@/types/ad";
```

The `packs` variable in the template comes directly from `usePacksList()` return value (a readonly ref), so `v-for="(item, index) in packs"` works without change.
