# Plan 055-03: Delete packs.store.ts and verify zero typecheck errors

## Goal

With all consumers migrated (Plans 055-01 and 055-02), delete `packs.store.ts` and confirm that `nuxt typecheck` passes with zero errors and no remaining imports reference the deleted file.

## Context

This plan runs after 055-01 and 055-02 complete. `packs/comprar.vue` still imports `usePacksStore` at module scope and will be deleted in Phase 56 — it must be stubbed here first (before the store is deleted) to keep `nuxt typecheck` passing. The stub is a typed placeholder; no behavioral change. After stubbing, the store file is deleted and typecheck is run to confirm zero errors.

## Tasks

<task id="1" title="Verify no remaining packs.store consumers (excluding comprar.vue)">

Before deleting, confirm all consumers are migrated. Run:

```bash
grep -rn "packs\.store\|usePacksStore" apps/website/app/ --include="*.vue" --include="*.ts" | grep -v "packs/comprar.vue" | grep -v "packs.store.ts"
```

Expected output: **empty** (zero matches). If any match appears, fix it before proceeding to deletion.

If only `packs/comprar.vue` appears in results (with the `-v` filter excluded), that is expected and acceptable — it will be cleaned up in Phase 56.

**Verification:** The grep command above produces zero lines of output (after excluding comprar.vue and the store file itself).
</task>

<task id="2" title="Stub comprar.vue, delete packs.store.ts, and run typecheck">

`packs/comprar.vue` imports `usePacksStore` at module scope. Deleting `packs.store.ts` will unconditionally break `nuxt typecheck` unless this import is neutralized first. The stub must happen **before** the deletion.

**Step 1 — Stub comprar.vue (mandatory, do this first):**

In `apps/website/app/pages/packs/comprar.vue`, remove the three packs.store lines. No replacement variable is needed — introducing an unused `ref` would trigger an ESLint unused-variable warning:

- Remove: `import { usePacksStore } from "@/stores/packs.store";`
- Remove: `const packsStore = usePacksStore();`
- Remove: any `await packsStore.loadPacks()` call

If `packsStore.packs` is referenced anywhere in the file (e.g. passed as a prop), replace that specific reference with `[]` inline — but do not introduce a named variable for it. The file is deleted entirely in Phase 56; the goal here is solely to prevent a broken import from blocking typecheck.

**Step 2 — Delete packs.store.ts:**
```bash
rm apps/website/app/stores/packs.store.ts
```

**Step 3 — Confirm deletion:**
```bash
ls apps/website/app/stores/packs.store.ts 2>&1
# Expected: "No such file or directory"
```

**Step 4 — Run typecheck:**
```bash
yarn workspace @waldo/website nuxt typecheck
```

Expected: zero TypeScript errors. If errors appear referencing `packs.store` in any file other than `comprar.vue`, it means a consumer was missed in Plans 055-01/055-02 — fix that consumer and re-run. Do NOT restore `packs.store.ts`.

**Verification:**
- `ls apps/website/app/stores/packs.store.ts` fails with "No such file or directory"
- `grep -n "packs.store" apps/website/app/pages/packs/comprar.vue` returns nothing (stub replaced it)
- `yarn workspace @waldo/website nuxt typecheck` exits with code 0
</task>
