# Plan 055-03: Delete packs.store.ts and verify zero typecheck errors

## Goal

With all consumers migrated (Plans 055-01 and 055-02), delete `packs.store.ts` and confirm that `nuxt typecheck` passes with zero errors and no remaining imports reference the deleted file.

## Context

This plan runs after 055-01 and 055-02 complete. It is purely subtractive — no new code, no refactors. The sole action is deletion of the store file and verification that nothing broke. The `packs/comprar.vue` page still imports `packs.store` but it is intentionally left in place for Phase 56 (which deletes that page entirely). Therefore the typecheck must be run after confirming `comprar.vue` is excluded from the "no remaining imports" check.

## Tasks

<task id="1" title="Verify no remaining packs.store consumers (excluding comprar.vue)">

Before deleting, confirm all consumers are migrated. Run:

```bash
grep -rn "packs\.store\|usePacksStore" apps/website/app/ --include="*.vue" --include="*.ts" | grep -v "packs/comprar.vue" | grep -v "packs.store.ts"
```

Expected output: **empty** (zero matches). If any match appears, fix it before proceeding to deletion.

If only `packs/comprar.vue` appears in results (with the `-v` filter excluded), that is expected and acceptable — it will be cleaned up in Phase 56.

**Verification:** The grep command above produces zero lines of output (after excluding comprar.vue and the store file itself).
</task>

<task id="2" title="Delete packs.store.ts and run typecheck">

**Delete the file:**
```bash
rm apps/website/app/stores/packs.store.ts
```

**Confirm deletion:**
```bash
ls apps/website/app/stores/packs.store.ts 2>&1
# Expected: "No such file or directory"
```

**Run typecheck:**
```bash
yarn workspace @waldo/website nuxt typecheck
```

Expected: zero TypeScript errors.

If typecheck reports errors referencing `packs.store`, it means a consumer was missed in Plans 055-01/055-02. Fix the consumer, then re-run typecheck. Do NOT restore `packs.store.ts`.

If typecheck reports errors in `packs/comprar.vue` due to the missing store, note them but do not fix them here — they will be resolved in Phase 56 (which deletes the page). However, TypeScript errors in that file WILL block the typecheck. In that case:

**Option A (preferred if comprar.vue errors appear):** Add a stub export to a temporary file, OR simply comment out the `usePacksStore` import in `comprar.vue` with a `// TODO Phase 56: delete this file` comment. Since `comprar.vue` is being deleted next phase, option A is adding a comment and empty stub is not necessary. The simplest approach: replace the `usePacksStore` import and usage in `comprar.vue` with a local `const packs = ref([])` placeholder — this is a one-line stub that will be deleted with the whole file in Phase 56.

**Verification:**
- `ls apps/website/app/stores/packs.store.ts` fails with "No such file or directory"
- `yarn workspace @waldo/website nuxt typecheck` exits with code 0
</task>
