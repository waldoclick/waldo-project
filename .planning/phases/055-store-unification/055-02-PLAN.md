# Plan 055-02: Migrate pages from packs.store to direct Strapi calls

## Goal

Migrate all four pages that import `packs.store` to use direct Strapi API calls inside `useAsyncData`, eliminating the store dependency without changing any page behavior.

## Context

Per AGENTS.md: `useAsyncData()` is the sole correct data-loading trigger in Nuxt pages. Each page currently wraps `packsStore.loadPacks()` inside `useAsyncData` but still reads `packsStore.packs` for the return value. The migration replaces the store-mediated load with a direct `strapi.find("ad-packs")` call, returning the data directly from `useAsyncData`. The Strapi SDK v5 cast pattern `response.data as unknown as Pack[]` must be applied.

`packs/gracias.vue` uses `getPackById` (filters by ID) — this becomes a direct Strapi `find` with a filter. That page also has `(data as any)` casts in the template — clean those up with typed access.

`anunciar/index.vue` is special: after removing `packsStore`, the `onMounted` analytics loop must iterate over the packs returned from `useAsyncData` (via a local `const packs` variable), not from the store.

**File 1:** `apps/website/app/pages/packs/index.vue`

Replace the `new Promise` wrapper anti-pattern with a clean async function:

```typescript
import type { Pack } from "@/types/pack";

const strapi = useStrapi();
const { data: packsData } = await useAsyncData<Pack[]>(
  "packs-page-packs",
  async () => {
    try {
      const response = await strapi.find("ad-packs", { populate: "*" });

      return [];
    }
  },
  { default: () => [] as Pack[] },
);
const packs = computed(() => packsData.value ?? []);
```

The `import type { Pack }` is already in the file — keep it. No other changes needed.

<task id="2" title="Migrate packs/gracias.vue">

**File:** `apps/website/app/pages/packs/gracias.vue`

This page uses `packsStore.getPackById(route.query.pack as string)` inside `useAsyncData`. Replace with a direct Strapi filter call:

```typescript
// BEFORE (inside useAsyncData)
const packsStore = usePacksStore();
const pack = await packsStore.getPackById(route.query.pack as string);
```

```typescript
// AFTER (inside useAsyncData)
const strapi = useStrapi();
const response = await strapi.find("ad-packs", {
  filters: { id: { $eq: route.query.pack } } as unknown as Record<string, unknown>,
  populate: "*",
} as unknown as Record<string, unknown>);
const pack = response.data?.[0] as unknown as Pack | undefined;
```

Remove:
- `import { usePacksStore } from "@/stores/packs.store";` (line 36)
- `const packsStore = usePacksStore();` (line 29 — top-level, not inside useAsyncData)

The `strapi` instance: use `const strapi = useStrapi();` at the top of the script block (before `useAsyncData`).

**Template cleanup — remove `(data as any)` casts:**

The template has three `(data as any).name`, `(data as any).price`, `(data as any).total_ads`. Replace with properly typed access. Since `data` is typed as `Pack | { error: string }`, add a typed local:

In the template, the condition `data && !('error' in data)` already narrows to `Pack`. However, Vue templates don't perform TypeScript narrowing. Use a computed:

```typescript
const packData = computed(() => {
  if (!data.value || "error" in data.value) return null;
  return data.value as Pack;
});
```

Then in the template replace:
- `(data as any).name` → `packData!.name`
- `(data as any).price` → `packData!.price`
- `(data as any).total_ads` → `packData!.total_ads`

And change the `v-if` condition from `data && !('error' in data)` to `packData`.

Also fix the `(pack as any).ads_count` cast in the `watch(data, ...)` callback:

The `pack` local variable is already typed as `Pack` in that callback. The `Pack` interface (in `pack.d.ts`) has `total_ads: number`, not `ads_count`. Replace `(pack as any).ads_count` with `pack.total_ads` on **both** occurrences in the watch callback:
- Line in `$setSEO` description: `(pack as any).ads_count` → `pack.total_ads`
- Line in `$setStructuredData` description: `(pack as any).ads_count` → `pack.total_ads`

The top-level `const packsStore = usePacksStore();` on line 29 is unused after the `useAsyncData` change (the store was only used inside `useAsyncData`). Remove it. Remove the import too.

The `useAsyncData` key is `"packData"` — this is not following the `'<page>-<data>'` naming convention. Change it to `"packs-gracias-pack"`.

**Verification:**
- `grep -n "packs.store\|packsStore\|as any" apps/website/app/pages/packs/gracias.vue` returns nothing
- `grep -n "packs-gracias-pack" apps/website/app/pages/packs/gracias.vue` confirms key rename
</task>

<task id="3" title="Migrate anunciar/index.vue">

**File:** `apps/website/app/pages/anunciar/index.vue`

This page has two `packs.store` dependencies:
1. `packsStore.loadPacks()` inside `useAsyncData("anunciar-init", ...)`
2. `packsStore.packs` inside `onMounted` for analytics

**Step 1 — Update `useAsyncData`:** Return the packs list from the async data call so it is available after SSR.

Replace the `useAsyncData` block:
```typescript
// BEFORE
await useAsyncData("anunciar-init", async () => {
  await Promise.all([
    meStore.loadMe(),
    categoriesStore.loadCategories(),
    packsStore.loadPacks(),
  ]);
  return true;
});
```

With:
```typescript
// AFTER
const strapi = useStrapi();
const { data: initData } = await useAsyncData("anunciar-init", async () => {
  const [, , packsResponse] = await Promise.all([
    meStore.loadMe(),
    categoriesStore.loadCategories(),
    strapi.find("ad-packs", { populate: "*" }),
  ]);
  return {
    packs: (packsResponse.data as unknown as Pack[]) ?? [],
  };
}, { default: () => ({ packs: [] as Pack[] }) });
```

**Step 2 — Update `onMounted` analytics loop:**

`initData` is a Nuxt `Ref<>` populated from SSR state transfer on client hydration — `.value` is safe to access in both `onMounted` and `watch` callbacks because Nuxt guarantees the value is present after hydration completes.

Replace `packsStore.packs` with `initData.value?.packs ?? []`:

```typescript
// BEFORE
for (const pack of packsStore.packs) {
```
```typescript
// AFTER
for (const pack of (initData.value?.packs ?? [])) {
```

Similarly in the `watch(() => adStore.pack, ...)` callback:
```typescript
// BEFORE
const selectedPack = packsStore.packs.find((p) => p.id === newPack);
```
```typescript
// AFTER
const selectedPack = (initData.value?.packs ?? []).find((p) => p.id === newPack);
```

**Step 3 — Remove packs.store imports:**
- Remove: `import { usePacksStore } from "@/stores/packs.store";`
- Remove: `const packsStore = usePacksStore();`

**Step 4 — Add Pack type import:**
Add `import type { Pack } from "@/types/pack";` with the other type imports.

**Step 5 — Confirm AnalyticsItem type:** The `pack.id` and `pack.price` accesses remain valid since `pack` is typed as `Pack` (has `id: number`, `price: number`, `total_ads: number`).

</task>
