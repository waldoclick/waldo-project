  <behavior>
    getAdStats(adDocumentId, days=14):
    - Test 1: given N ad-view rows spread across the window → returns `{ total: N, series: number[] }` where series.length === days and sum(series) === total (each bucket = count of views whose viewed_at falls on that calendar day).
    - Test 2: series is ordered oldest→newest, the last element being today's bucket.
    - Test 3: conversion = round(contacts / views * 100) when views>0, else 0 — returns `conversion` as an integer percent; also returns `contacts` (count of ad-contact rows for the ad) and `avgPerDay = round(total/days)`.
    - Test 4: ad with zero views → `{ total:0, series:[0,...x days], contacts:0, conversion:0, avgPerDay:0 }` (no division by zero).

    getUserTotalViews(userId):
    - Test 5: sums ad-view counts only over ads belonging to userId that are active → returns the integer total (delegates the "active ads" filter to a query on api::ad.ad by user + active=true, then counts views for those ad ids).
  </behavior>
  <implementation>
    1. Service `getAdStats(adDocumentId, days = 14)`:
       - Resolve ad by documentId → ad.id. If missing → return zeros shape.
       - Compute window start = today minus (days-1) at 00:00. Build `days` empty buckets keyed by yyyy-mm-dd (oldest→newest).
       - `findMany` ad-view rows where `ad = ad.id` AND `viewed_at >= windowStart`; for each, bucket by `viewed_at` calendar day. total = rows.length (or count all-time views — DECIDE: "Vistas totales" in the modal is all-time total per mockup line 484; the 14-day series is the chart. So total = count of ALL ad-view rows for the ad; series = last `days` buckets). Implement both: `total` = count(all ad-view for ad); `series` = counts within the window.
       - contacts = count(ad-contact where ad = ad.id).
       - conversion = views>0 ? Math.round(contacts/views*100) : 0 (views = total).
       - avgPerDay = Math.round(total/days) (mockup line 489 "Prom. X / día").
       - Return `{ total, series, contacts, conversion, avgPerDay }`.
    2. Service `getUserTotalViews(userId)`:
       - Query api::ad.ad where `user = userId` AND `active = true` → collect ad ids.
       - count ad-view rows where `ad in adIds`. Return integer. (Use strapi.db.query count.)
    3. Controller actions:
       - `stats(ctx)`: read `ctx.params.documentId`, `ctx.query.days` (default 14, clamp 1..90) → `getAdStats`. (Ownership: this is the owner's stats — require auth; verify the ad belongs to ctx.state.user.id or is manager, else forbidden. Reuse the ctxIsManager pattern from ad controller.)
       - `panelViewsTotal(ctx)`: read `ctx.state.user.id` (auth required) → `getUserTotalViews`. Return `{ total }`.
    4. Routes in 00-ad-view-custom.ts:
       - `{ method:"GET", path:"/ads/:documentId/stats", handler:"ad-view.stats" }` (auth on — owner only).
       - `{ method:"GET", path:"/ads/me/views-total", handler:"ad-view.panelViewsTotal" }` (auth on). Declare the static `/ads/me/views-total` BEFORE `/ads/:documentId/stats` so the wildcard doesn't capture "me".
    5. Test `apps/strapi/tests/api/ad-view/ad-view.stats.test.ts`: AAA, mock strapi.db.query with per-UID routing (ad lookup, ad-view findMany/count, ad-contact count). Assert series length, sum===total-within-window, conversion math, zero-division guard, and getUserTotalViews active-ads filtering.
  </implementation>
</feature>
