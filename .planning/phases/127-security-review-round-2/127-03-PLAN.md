<objective>
Close SEC2-AUTH (four fixes):
1. `google-one-tap.service.ts findOrCreateUser` links a Google identity to a local account by email match WITHOUT checking `payload.email_verified === true` — a pre-hijack vector in federated/Workspace domains.
2. `ad.ts:757` uses `process.env.JWT_SECRET ?? "strapi-jwt-secret"` — if JWT_SECRET is empty, tokens can be forged.
3. Auth endpoints have NO rate limiting (only reCAPTCHA in the proxy) — password spraying / email bombing.
4. reCAPTCHA verification ignores `action`/`hostname` (`google-recaptcha.service.ts:22` and `apps/website/server/utils/recaptcha.ts`) — token replay across forms.

Purpose: harden authentication against account pre-hijack, token forgery, brute force, and CAPTCHA replay.
Output: email_verified guard, plugin-JWT verification, two-layer rate limiting, reCAPTCHA action+hostname binding, and regression tests.
</objective>

    NUXT PROXY LAYER — create `apps/website/server/middleware/auth-rate-limit.ts`:
    ```typescript
    const AUTH_RATE_PATHS = [
      "/api/auth/local",
      "/api/auth/local/register",
      "/api/auth/forgot-password",
      "/api/auth/verify-code",
      "/api/auth/resend-code",
      "/api/auth/google-one-tap",
    ];

    // NOTE: single-process in-memory — PM2 cluster mode needs a shared store
    const ipStore = new Map<string, { count: number; windowStart: number }>();
    const WINDOW_MS = 60_000;
    const MAX_REQUESTS = 10;

    export default defineEventHandler((event) => {
      const path = event.node.req.url ?? "";
      const isAuthPath = AUTH_RATE_PATHS.some((p) => path.startsWith(p));
      if (!isAuthPath || event.node.req.method === "GET") return;

      const ip =
        getHeader(event, "x-forwarded-for")?.split(",")[0]?.trim() ??
        event.node.req.socket?.remoteAddress ??
        "unknown";

      const now = Date.now();
      const entry = ipStore.get(ip);
      if (!entry || now - entry.windowStart > WINDOW_MS) {
        ipStore.set(ip, { count: 1, windowStart: now });
        return;
      }
      entry.count += 1;
      if (entry.count > MAX_REQUESTS) {
        setResponseStatus(event, 429);
        return { statusCode: 429, statusMessage: "Too Many Requests" };
      }
    });
    ```
