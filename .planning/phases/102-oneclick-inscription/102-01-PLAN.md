         - **Store token on user record:** `await strapi.entityService.update("plugin::users-permissions.user", user.id, { data: { pro_inscription_token: String(result.token) } as unknown as Parameters<typeof strapi.entityService.update>[2]["data"] })` — this maps the token to the user for the finish handler

         - This is a GET handler — Transbank redirects here after card enrollment
         - Read `TBK_TOKEN` from `ctx.query.TBK_TOKEN`
         - **Cancelled case:** If `!TBK_TOKEN`, redirect to `${process.env.FRONTEND_URL}/pagar/error?reason=cancelled` and return

         - **Rejected case:** If `!result.success || !result.tbkUser`, redirect to `${process.env.FRONTEND_URL}/pagar/error?reason=rejected` and return
         - **Success case:** Resolve user by pro_inscription_token lookup:
           - `const user = await strapi.db.query("plugin::users-permissions.user").findOne({ where: { pro_inscription_token: String(TBK_TOKEN) } })`
           - If no user found, redirect to `${process.env.FRONTEND_URL}/pagar/error?reason=rejected` and return
         - Update user record with `strapi.entityService.update("plugin::users-permissions.user", user.id, { data: { pro: true, pro_status: "active", tbk_user: result.tbkUser, pro_card_type: result.cardType, pro_card_last4: result.last4CardDigits, pro_expires_at: new Date(Date.now() + 30 * 24 * 60 * 60 * 1000), pro_inscription_token: null } as unknown as Parameters<typeof strapi.entityService.update>[2]["data"] })`
         - Redirect to `${process.env.FRONTEND_URL}/pro/gracias?inscribed=true`
