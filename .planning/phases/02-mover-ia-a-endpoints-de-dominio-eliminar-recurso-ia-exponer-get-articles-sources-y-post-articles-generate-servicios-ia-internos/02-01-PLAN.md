<objective>
Build the domain-side AI surface inside the `article` API, additively. After this plan the new
endpoints `GET /articles/sources` and `POST /articles/generate` exist and work, the AI provider is
selected and prompt is built on the backend, and a fallback chain handles provider failures. The old
`ia` and `search` resources are left untouched here — they are removed in Plan 02 once the frontend
is cut over.

Purpose: Move the AI from its own `ia` resource to domain endpoints (anti-pattern correction) while
keeping the dashboard news flow behavior identical (D-04, D-05, D-06, D-07).
Output: A new internal `ai-provider` orchestrator service, lazy-loaded AI service index files,
and two new manager-only article actions + routes.
</objective>

The AI service index files today eagerly instantiate at module top-level, so importing them THROWS
when the API key is missing. Only cerebras is lazy. This must be fixed for the fallback chain to work.

Current cerebras/index.ts (the LAZY pattern to mirror in the other 4):
```typescript
import { CerebrasService } from "./cerebras.service";
let cerebrasService: CerebrasService | null = null;
function getCerebrasService(): CerebrasService {
  if (!cerebrasService) {
    cerebrasService = new CerebrasService();
  }
  return cerebrasService;
}
export const generateText = (prompt: string) =>
  getCerebrasService().generate({ prompt });
export { CerebrasService };
export type { ICerebrasService, CerebrasRequest, CerebrasResponse } from "./cerebras.types";
```

Current EAGER index files to convert (all instantiate `new XService()` at top-level):
- gemini/index.ts    → exports `generateText`, class `GeminiService`
- groq/index.ts      → exports `generateText`, class `GroqService`
- deepseek/index.ts  → exports `generateText`, class `DeepSeekService`
- anthropic/index.ts → exports `generateWithSearch` (NOTE the different name), class `AnthropicService`

Each service exposes the same shape: `generate({ prompt }): Promise<{ text }>` and constructor throws
`Error("<PROVIDER>_API_KEY environment variable is required")` when its key is missing.
Env var names: GEMINI_API_KEY, GROQ_API_KEY, DEEPSEEK_API_KEY, ANTHROPIC_API_KEY, CEREBRAS_API_KEY.

Tavily service (already on the backend — the source of `/articles/sources`):
```typescript
// apps/strapi/src/services/tavily/index.ts
export const searchNews = (query: string, num?: number) => ...;
// returns: { news: TavilyNewsResult[] }
// TavilyNewsResult = { title, link, snippet, date, source, imageUrl? }
```

Old ia controller logic being replaced (apps/strapi/src/api/ia/controllers/ia.ts) — each handler did:
read `body.prompt`, call the provider's `generate`, return `{ text: result.text }`. The cerebras
handler is the one the front uses today.

The prompt template currently lives in the FRONTEND
(apps/website/app/components/LightBoxArticles.vue, const DEFAULT_GEMINI_PROMPT). It must move here.

Controller extension pattern (template = apps/strapi/src/api/ad/controllers/ad.ts):
```typescript
export default factories.createCoreController("api::article.article", ({ strapi }) => ({
  async sources(ctx) { ... },
  async generate(ctx) { ... },
}));
```

Custom-route ordering convention (template = apps/strapi/src/api/ad/routes/00-ad-custom.ts):
a file prefixed `00-` loads BEFORE the core router `article.ts`, so the static `/articles/sources`
path registers before the core `GET /articles/:id` wildcard. Without this the wildcard captures
`sources` as an id.

isManager policy: `global::isManager` (apps/strapi/src/policies/isManager.ts) — true when
ctx.state.user.role.name === "manager" (case-insensitive).
