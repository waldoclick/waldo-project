<objective>
Migrate all `strapi.find()` and `strapi.findOne()` read calls in the dashboard store and all 29 components to use `useApiClient()` instead.

Purpose: Eliminate the dual-resource pattern (useStrapi for reads, useApiClient for writes) so all HTTP goes through one composable. This is the component/store half of the migration.
Output: 30 files updated (1 store + 29 components) with zero remaining `strapi.find`/`strapi.findOne` calls.
</objective>

From apps/dashboard/app/composables/useApiClient.ts:
```typescript
export function useApiClient() {
  // Returns: async function apiClient<T = unknown>(url: string, options?: Parameters<typeof client>[1]): Promise<T>
  // For GET: apiClient(url, { method: "GET", params: {...} })
  // For mutations: automatically injects X-Recaptcha-Token
}
```

Migration patterns (from RESEARCH.md):
- `strapi.find(collection, queryParams)` -> `apiClient(collection, { method: "GET", params: queryParams as unknown as Record<string, unknown> }) as { data: T[]; meta: { pagination: Pagination } }`
- `strapi.find("users/me", params)` -> `apiClient("users/me", { method: "GET", params: params as unknown as Record<string, unknown> })` (returns user directly, NOT wrapped in { data: T })
- `strapi.find("indicators/dashboard-stats" as any, {} as any)` -> `apiClient("indicators/dashboard-stats", { method: "GET" }) as { data: T }`
- `strapi.find("orders/sales-by-month" as any, { year } as any)` -> `apiClient("orders/sales-by-month", { method: "GET", params: { year } as unknown as Record<string, unknown> }) as { data: T[] }`
