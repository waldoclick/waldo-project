<interfaces>
<!-- Contract for InputPhone (created in 124-01) -->
```typescript
// Props
interface InputPhoneProps {
  modelValue: string;  // combined value like "+56912345678"
}

// Emits
// 'update:modelValue' with combined dialCode + localNumber string
```

Usage pattern (replaces bare <Field> for phone):
```html
<div class="form-group">
  <label class="form-label" for="phone">Teléfono *</label>
  <Field v-slot="{ field }" name="phone">
    <InputPhone v-bind="field" />
  </Field>
  <ErrorMessage name="phone" />
</div>
```

This keeps vee-validate's `name="phone"` registered against the form's yup schema, while `InputPhone` handles the composite UI and emits a combined string that yup validates exactly like the old bare input.

Yup rule already in each form (DO NOT CHANGE):
```typescript
phone: yup
  .string()
  .required('Teléfono es requerido')
  .min(11, 'El teléfono debe tener al menos 11 caracteres')
  .max(20, 'El teléfono no puede exceder los 20 caracteres')
  .matches(/^[\d\s()+-]+$/, '...')
```
Note: FormContact.vue's rule is optional (no `.required()`).
