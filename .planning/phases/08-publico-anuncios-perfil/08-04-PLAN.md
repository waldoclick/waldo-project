<objective>
Build the shared Strapi backend for the obfuscated-contact + separate-reveal model that BOTH the ad detail (08-02) and the seller profile (08-03) consume, plus the seller's Vendidos endpoint. This is the ONLY plan touching apps/strapi in phase 08 — 08-01/08-02/08-03 are website-only. Wave 1, no website dependency, so it runs parallel to the 08-01 card plan.

Net-new, user-approved contact architecture:
1. Bulk payloads (ad detail + seller profile) return seller email/phone/whatsapp OBFUSCATED + presence flags — the REAL values stop travelling in payloads.
2. FIVE single-purpose reveal endpoints return the REAL value of ONE channel to a logged-in viewer only: three ad-keyed (phone/whatsapp/email, which also record an ad-contact event) and two seller-keyed (phone/whatsapp, no recording).
3. A public soldAds endpoint for the profile Vendidos tab.

Purpose: contact data is the conversion surface AND a PII/anti-scraping surface. Obfuscating the bulk payload kills bot scraping while single-purpose reveal endpoints expose the real value only to authenticated humans at the moment they contact — honest, no N+1.
Output: contact-mask helpers + Jest, obfuscation wired into both sanitize paths, 5 reveal endpoints + Jest, soldAds service + route + Jest.
</objective>
