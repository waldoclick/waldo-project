fixes both SEO signals with zero user-visible changes.

<!-- Executor must understand WHY LocaleObject is required, not just string[] -->

From nuxt-site-config `runtime/app/plugins/i18n.js`:
```javascript
const defaultLocale = computed(() => {
  const locale = toValue(i18n.locales).find((l) => l.code === i18n.defaultLocale);
  return locale?.language || locale?.iso || i18n.defaultLocale;
  // With locales: ['es']  → returns 'es'       → og:locale = 'es' (no underscore = NOT set)
  // With locales: [{ code: 'es', language: 'es-ES' }] → returns 'es-ES' → og:locale = 'es_ES' ✓
});
```

From nuxt-seo-utils `runtime/app/logic/applyDefaults.js`:
```javascript
ogLocale: () => {
  const locale = resolveCurrentLocale(); // e.g. 'es-ES'
  if (locale) {
    const l = locale.replace("-", "_");  // 'es_ES'
    if (l.includes("_")) {
      return l; // ← only set when it contains underscore
    }
  }
  return false; // ← og:locale NOT set if locale has no region tag
},
htmlAttrs: { lang: resolveCurrentLocale }, // 'es-ES' → lang="es-ES"
```

@nuxtjs/i18n v9.5.5 config shape (from module.d.ts):
```typescript
interface NuxtI18nOptions {
  defaultLocale?: string;            // 'es'
  locales?: string[] | LocaleObject[]; // use LocaleObject[] for og:locale
  strategy?: 'no_prefix' | 'prefix_except_default' | 'prefix' | 'prefix_and_default';
  detectBrowserLanguage?: DetectBrowserLanguageOptions | false;
  langDir?: string | null;           // relative to app/ dir → 'locales'
  bundle?: {
    optimizeTranslationDirective?: boolean; // set false to suppress deprecation warning
  };
}

// LocaleObject for proper og:locale support:
interface LocaleObject {
  code: string;      // 'es'
  language: string;  // 'es-ES' → becomes og:locale = 'es_ES'
  name?: string;     // 'Español'
}
```

    Using `{ code: 'es', language: 'es-ES' }` produces `og:locale = es_ES`.

    **Why `no_prefix` strategy:** This is a single-language app. No URL prefixes
    (`/es/anuncios`) needed. The sitemap uses unprefixed URLs already.

    **Why `detectBrowserLanguage: false`:** Only one locale — detection would redirect
    and serve cookies with no functional benefit.
