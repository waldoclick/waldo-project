  <action>
Apply the **array-length + timestamp** cache guard pattern (from `categories.store.ts`) to all three stores. Use the Options API style since these stores use `defineStore("id", { state, actions })` — not the Composition API setup function.

**Pattern to apply (Options API translation):**
- Add `lastFetch: 0` to `state()`
- At the top of the load action, compute `cacheAge = Date.now() - this.lastFetch`
- If data already loaded AND cache is fresh → return early (skip HTTP call)
- After a successful fetch → set `this.lastFetch = Date.now()`
- Cache duration: 30 minutes (1800000 ms) — consistent with categories.store.ts

**packs.store.ts** — add to `state`: `lastFetch: 0`. Add guard at top of `loadPacks()`:
```
const now = Date.now();
if (this.packs.length > 0 && now - this.lastFetch < 1800000) return;
```
After `this.packs = response.data as unknown as Pack[]`, add: `this.lastFetch = Date.now();`
Note: packs.store.ts has no `persist` — add it with localStorage storage (same pattern as conditions/regions). This ensures packs survive page refresh just like the other reference data stores.

**conditions.store.ts** — add to `state`: `lastFetch: 0`. Add guard at top of `loadConditions()`, BEFORE the `try` block:
```
const now = Date.now();
if (this.conditions.length > 0 && now - this.lastFetch < 1800000) return;
```
After `this.conditions = typedResponse.data;`, add: `this.lastFetch = Date.now();`

**regions.store.ts** — add to `state`: `lastFetch: 0`. Add guard at top of `loadRegions()`, BEFORE the `try` block:
```
const now = Date.now();
if (this.regions.data.length > 0 && now - this.lastFetch < 1800000) return;
```
After `this.regions = typedResponse;`, add: `this.lastFetch = Date.now();`

**Do NOT touch:** `getPackById()` and `loadConditionById()` — these are on-demand lookups by ID, not list fetches. Cache guard only applies to the list-loading actions.

**Update typed state interfaces** — `conditions.store.ts` and `regions.store.ts` use typed state interfaces that must include `lastFetch` or TypeScript will reject the new field:

- `apps/website/app/types/condition.d.ts`: Add `lastFetch: number;` to `ConditionState`:


Note: `packs.store.ts` does NOT use a typed state interface (its `state()` returns an inline object literal), so no type file update is needed for it.

**Do NOT fix pre-existing LSP errors** in these files. Only add the cache guard lines.
