**apps/strapi/src/api/news/content-types/news/schema.json**
```json
{
  "kind": "collectionType",
  "collectionName": "news_items",
  "info": {
    "singularName": "news",
    "pluralName": "news",
    "displayName": "News",
    "description": "News articles for the platform"
  },
  "options": {
    "draftAndPublish": true
  },
  "pluginOptions": {},
  "attributes": {
    "title": {
      "type": "string",
      "required": true
    },
    "header": {
      "type": "string",
      "required": false
    },
    "body": {
      "type": "richtext",
      "required": false
    },
    "cover": {
      "type": "media",
      "multiple": true,
      "required": false,
      "allowedTypes": ["images"]
    },
    "gallery": {
      "type": "media",
      "multiple": true,
      "required": false,
      "allowedTypes": ["images"]
    },
    "categories": {
      "type": "relation",
      "relation": "manyToMany",
      "target": "api::category.category"
    },
    "seo_title": {
      "type": "string",
      "required": false
    },
    "seo_description": {
      "type": "string",
      "required": false
    }
  }
}
```

Key decisions:
- `"draftAndPublish": true` — enables Strapi's native draft/publish toggle (NEWS-03). No custom `status` field.
- `cover` and `gallery` are both `media, multiple: true, allowedTypes: ["images"]` (NEWS-01).
- `body` uses `"type": "richtext"` for Strapi's rich text editor (NEWS-01).
- `categories` is `manyToMany` with `api::category.category` — optional (NEWS-02). Many-to-many allows a news entry to belong to multiple categories and vice versa.
- `seo_title` and `seo_description` are `"type": "string"` (short text), both optional (NEWS-08).
- `collectionName: "news_items"` avoids collision with SQL reserved word `news` in some databases.
