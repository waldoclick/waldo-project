  <name>Task 1: Define IWebpayReceipt interface</name>
  <files>apps/website/app/types/webpay-receipt.d.ts</files>
  <action>
Create TypeScript interface for Webpay receipt data structure. Include all mandatory fields per Chilean/Webpay standards:

```typescript
export interface IWebpayReceipt {
  // Required fields (all optional to handle missing data gracefully)
  amount?: number;              // Monto
  authorizationCode?: string;   // Código de autorización
  transactionDate?: string;     // Fecha y hora de pago (ISO format)
  paymentType?: string;         // Tipo de pago (e.g., "VD", "VN")
  cardNumber?: string;          // Últimos 4 dígitos
  buyOrder?: string;            // Número de orden/compra
  commerceCode?: string;        // Código de comercio
  status?: string;              // Estado (e.g., "AUTHORIZED")
}
```

Map to actual Webpay API fields (from payment_response JSON):
- `amount` → response.amount
- `authorizationCode` → response.authorization_code
- `transactionDate` → response.transaction_date
- `paymentType` → response.payment_type_code
- `cardNumber` → response.card_detail?.card_number (last 4)
- `buyOrder` → response.buy_order
- `commerceCode` → response.commerce_code
- `status` → response.status

All fields optional because API may return incomplete data — component must handle gracefully with placeholders.

Use PascalCase with `I` prefix per AGENTS.md naming conventions.
