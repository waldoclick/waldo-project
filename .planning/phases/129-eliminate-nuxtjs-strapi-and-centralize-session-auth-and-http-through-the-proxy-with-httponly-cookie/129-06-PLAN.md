<objective>
CUTOVER. In one coordinated wave: batch-rename all remaining `useStrapiUser`/`useStrapiAuth` consumers to `useSessionUser`/`useSessionAuth`, swap `useApiClient` onto `useSessionClient`, activate the `session.ts` plugin, remove `@nuxtjs/strapi` from `nuxt.config.ts` modules + `package.json`, remove the `runtimeConfig.strapi.url=API_URL` hack and the top-level `strapi:` module block, update test mocks, run a final grep sweep proving zero `@nuxtjs/strapi` / `useStrapiX` / direct-API_URL / direct-fetch remain, and verify every auth flow end-to-end.

Why all-at-once (per advisor): the module's auto-fetch plugin and the API_URL hack are coupled liabilities. Removing the hack early reroutes the OLD plugin's SSR fetchUser onto an unauthenticated path → reintroduces the logout bug. Activating session.ts while the module's plugin still runs → split-brain (two fetchUser writers). So module removal + hack removal + session.ts activation + the client swap MUST land together.

Purpose: Complete the migration and make the system coherent (it is knowingly half-migrated until this plan).
Output: Module-free, useStrapiX-free website; active session plugin; passing tests; verified auth flows.
</objective>
