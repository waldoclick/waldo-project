<objective>
Make `useApiClient` SSR-correct for the httpOnly architecture (forward the browser cookie jar so the proxy can read `waldo_jwt`; inject the Vercel protection-bypass header; stop injecting `X-Proxy-Key` because the catch-all proxy now owns it), wire the `vercelBypassSecret` private runtimeConfig key, and add the one-line `?json=true` branch to the Strapi `auth-google` callback so the popup OAuth Nitro route can read `{ jwt }` as JSON.

SAFE in wave 1: `useApiClient` keeps wrapping `useStrapiClient` for now (the swap to `useSessionClient` is the cutover in plan 06). The SSR header additions are harmless while the module runs ‚Äî they only help the proxy and Vercel. The Strapi change is additive and backward-compatible (default HTML path unchanged).

Purpose: Close the SSR cookie-forwarding gap (Pitfall 3) and unblock the popup OAuth JSON exchange (Pitfall 9) before consumer migration.
Output: SSR-aware useApiClient, runtimeConfig key, backward-compatible Strapi JSON branch, updated/added tests.
</objective>
