<objective>
Build the server-side foundation: (1) modify the catch-all proxy so it injects `Authorization: Bearer <jwt>` from the httpOnly cookie (instead of forwarding a `Cookie` header that Strapi ignores) and stops forwarding client-supplied Authorization; (2) create the dedicated Nitro auth intercept routes that consume the JWT-issuing Strapi responses and set the httpOnly `waldo_jwt` cookie server-side, never returning the JWT to the client.

These are SAFE to land in wave 1: the proxy change only affects requests that already carry the `waldo_jwt` cookie (set today by the module), and the new routes are PASSIVE — no client calls them until plan 04/05 wires consumers. The `@nuxtjs/strapi` module keeps running undisturbed.

Purpose: Establish the single server-side exit point + httpOnly token issuance before any consumer migration.
Output: Modified proxy, 6 new server routes, 1 server-route test.

IMPORTANT — two-step login correction (verified against live code, NOT the research's generic example):
`/auth/local` returns `{ pendingToken, email }` — NO JWT. Do NOT create a `login.post.ts` intercept. `/auth/local` stays on the catch-all (keeps its method-based reCAPTCHA automatically). The JWT for email/password login is issued by `/auth/verify-code` — THAT is the intercept route.
</objective>

```typescript
import { verifyRecaptchaToken } from "../../utils/recaptcha";

export default defineEventHandler(async (event) => {
  const config = useRuntimeConfig(event);

  // reCAPTCHA — this route bypasses the catch-all's reCAPTCHA (Pitfall 10).
  if (config.recaptchaEnabled) {
    const recaptchaToken = getHeader(event, "x-recaptcha-token");
    await verifyRecaptchaToken(recaptchaToken, config.recaptchaSecretKey as string);
  }

  const apiUrl = process.env.API_URL || "http://localhost:1337";
  const body = await readBody(event);

  const strapiResponse = await $fetch<{ jwt: string; user: unknown }>(
    `${apiUrl}/api/auth/verify-code`,
    {
      method: "POST",
      body,
      headers: { "X-Proxy-Key": config.proxySecretWeb as string },
    },
  );

  setCookie(event, "waldo_jwt", strapiResponse.jwt, {
    httpOnly: true,
    secure: process.env.NODE_ENV === "production",
    sameSite: "lax",
    path: "/",
    domain: process.env.COOKIE_DOMAIN || undefined,
    maxAge: 604800,
  });

  return { user: strapiResponse.user }; // jwt NEVER returned to client
});
```

```typescript
import { verifyRecaptchaToken } from "../../../utils/recaptcha";

export default defineEventHandler(async (event) => {
  const config = useRuntimeConfig(event);

  // reCAPTCHA — this JWT-issuing route bypasses the catch-all's reCAPTCHA (Pitfall 10).
  if (config.recaptchaEnabled) {
    const recaptchaToken = getHeader(event, "x-recaptcha-token");
    await verifyRecaptchaToken(recaptchaToken, config.recaptchaSecretKey as string);
  }

  const apiUrl = process.env.API_URL || "http://localhost:1337";
  const { access_token } = await readBody<{ access_token: string }>(event);

  const result = await $fetch<{ jwt: string; user: unknown }>(
    `${apiUrl}/api/auth/google/callback?access_token=${encodeURIComponent(access_token)}`,
    { headers: { "X-Proxy-Key": config.proxySecretWeb as string } },
  );

  setCookie(event, "waldo_jwt", result.jwt, {
    httpOnly: true,
    secure: process.env.NODE_ENV === "production",
    sameSite: "lax",
    path: "/",
    domain: process.env.COOKIE_DOMAIN || undefined,
    maxAge: 604800,
  });

  return { user: result.user };
});
```
