<objective>
Close the HIGH account-takeover IDOR on `PUT /api/users/:id`. Today any authenticated user can PUT an arbitrary user id and modify that user's record (e.g. email/password) because the built-in users-permissions `user.update` route has no ownership check. Add an ownership policy that rejects (403) any request where `ctx.params.id !== ctx.state.user.id`, with managers bypassing via the existing `global::isManager` helper, and register it on the built-in `user.update` route.

Purpose: Prevent cross-user account takeover while preserving every legitimate self-service flow.
Output: A new `is-users-owner` policy, its registration in `strapi-server.ts`, and a Jest regression test.
</objective>

  <action>
    Create a Strapi v5 policy file that default-exports a function `(policyContext, _config, { strapi })`.

    Logic:
    1. Read the authenticated user: `const user = policyContext.state.user;`
    2. If there is no authenticated user → return false (denies; Strapi yields 403).
    3. Determine manager bypass: `const roleName = (user?.role?.name ?? "").toLowerCase(); const isManager = roleName === "manager";` If `isManager` → return true (managers may update any user).
    4. Read the path id: `const paramId = policyContext.params?.id;` (the request path `:id`).
    5. Compare ownership string-safe: `return String(paramId) === String(user.id);` Return true only when the caller owns the record; false otherwise (→ 403).

    Use this exact signature so Strapi treats it as a policy:
    ```ts
    import type { Core } from "@strapi/strapi";

    export default (
      policyContext: {
        state: { user?: { id: number | string; role?: { name?: string } } };
        params: { id?: string | number };
      },
      _config: Record<string, unknown>,
      _deps: { strapi: Core.Strapi },
    ): boolean => {
      const user = policyContext.state.user;
      if (!user) return false;
      const roleName = (user.role?.name ?? "").toLowerCase();
      if (roleName === "manager") return true;
      return String(policyContext.params?.id) === String(user.id);
    };
    ```
    Do NOT leave any unused variable or import (Codacy rule). If `_deps`/`strapi` is unused, keep the `_`-prefixed param only if it is required by the policy signature; otherwise omit it. Code/comments in English.
