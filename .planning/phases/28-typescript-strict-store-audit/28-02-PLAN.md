  <action>
Add an inline comment on the line immediately above the `persist:` key in each store.
The comment format is: `// persist: <TAG> — <one-line rationale>`

Tags: CORRECT (intentional, appropriate), REVIEW (potentially volatile or partially appropriate),
RISK (stale data is likely, persistence probably should not survive page reload).

Do NOT change any logic, types, or existing comments. Only add the one new comment line.
Do NOT add emojis. Comments in English.

Prescribed classification for each store (use exactly these):

**ads.store.ts** — RISK
```
// persist: RISK — query results (ads[], pagination) are volatile; stale listings survive reload and may show deleted/outdated ads
```

**ad.store.ts** — CORRECT
```
// persist: CORRECT — multi-step ad creation wizard; form data must survive page refresh between steps
```

**app.store.ts** — REVIEW
```
// persist: REVIEW — referer and contactFormSent are session-scoped; isMobileMenuOpen is volatile UI state that should reset on reload
```

**categories.store.ts** — CORRECT
```
// persist: CORRECT — static reference data with 30-min cache TTL (lastFetch); safe to reuse across sessions
```

**communes.store.ts** — REVIEW
```
// persist: REVIEW — static reference data but no TTL; communes are stable in practice but could go stale indefinitely
```

**conditions.store.ts** — CORRECT
```
// persist: CORRECT — static reference data with 30-min cache TTL (lastFetch); safe to reuse across sessions
```

**faqs.store.ts** — CORRECT
```
// persist: CORRECT — static reference data with 1-hour cache TTL (lastFetchTimestamp); safe to reuse across sessions
```

**filter.store.ts** — CORRECT
```
// persist: CORRECT — static reference data with 30-min cache TTL (lastFetchCommunes, lastFetchCategories); safe to reuse across sessions
```

**history.store.ts** — CORRECT
```
// persist: CORRECT — explicit user browsing history feature; must survive reload to show recently viewed ads
```

**indicator.store.ts** — CORRECT
```
// persist: CORRECT — daily economic indicators with date-based cache (lastFetchDate); valid to reuse on same calendar day
```

**packs.store.ts** — CORRECT
```
// persist: CORRECT — static reference data with 30-min cache TTL (lastFetch); safe to reuse across sessions
```

**pack.store.ts** — REVIEW
```
// persist: REVIEW — pack selection and invoice flag are ad creation wizard state; overlaps with ad.store; persisting separately risks inconsistency
```

**regions.store.ts** — CORRECT
```
// persist: CORRECT — static reference data with 30-min cache TTL (lastFetch); safe to reuse across sessions
```

**related.store.ts** — RISK
```
// persist: RISK — related ads are specific to the last-viewed ad; stale results appear on unrelated ad pages after reload
```

Place the comment on the line immediately before `persist:` in each store file.
