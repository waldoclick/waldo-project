  <name>Task 2: Add success emit to FormProfile (backward-compatible)</name>
  <files>apps/website/app/components/FormProfile.vue</files>
  <action>
Modify `FormProfile.vue` to emit a `success` event after save, while preserving backward compatibility with `AccountEdit.vue`.

1. At the top of `<script setup>`, add:
   ```
   const emit = defineEmits(["success"]);
   const props = defineProps({ onboardingMode: { type: Boolean, default: false } });
   ```

2. In the `handleSubmit` function, find the line (currently line 687):
   ```
   window.location.href = "/cuenta/perfil";
   ```
   Replace it with:
   ```
   emit("success");
   if (!props.onboardingMode) {
     window.location.href = "/cuenta/perfil";
   }
   ```

This approach:
- When `onboardingMode` is NOT passed (AccountEdit.vue): emits success (ignored, no listener), then does the hard redirect as before. Backward compatible.
- When `onboardingMode` is true (OnboardingDefault.vue in Plan 02): emits success (parent handles navigation), does NOT do the hard redirect.

Do NOT use `getCurrentInstance()?.vnode.props?.onSuccess` introspection — it is fragile per research recommendations. The `onboardingMode` prop approach is simpler and explicit.
  </action>

  </verify>
  <done>FormProfile.vue has `defineEmits(["success"])` and `defineProps({ onboardingMode })`. When `onboardingMode` is false (default), the existing redirect to /cuenta/perfil fires. When true, only the emit fires.</done>
