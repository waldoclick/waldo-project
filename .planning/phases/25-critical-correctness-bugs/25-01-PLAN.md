  <name>Task 1: BUG-01 — Add type declaration to microdata.ts plugin</name>
  <files>apps/website/app/plugins/microdata.ts</files>
  <action>
    The plugin correctly provides `$setStructuredData` via `return { provide: { setStructuredData } }`
    but has no `declare module "#app"` type augmentation. This causes pages to use fragile
    `as unknown as { $setStructuredData }` casts.

    Append the following block at the end of the file (after the closing `});`):

    ```typescript
    declare module "#app" {
      interface NuxtApp {
        $setStructuredData: (data: object) => void;
      }
    }
    ```

    The plugin body itself is CORRECT — do not modify any existing code. Only append the declaration.

    Reference: `apps/website/app/plugins/seo.ts` already uses this identical pattern correctly.
    Note: Nuxt automatically prefixes `provide` keys with `$` — so `provide: { setStructuredData }`
    becomes `$setStructuredData` at runtime. The type declaration must use the `$`-prefixed name.
  </action>
  <verify>
    <automated>grep -n "declare module" apps/website/app/plugins/microdata.ts</automated>
  </verify>
  <done>
    File contains `declare module "#app" { interface NuxtApp { $setStructuredData: (data: object) => void; } }`.
    Any page that calls `useNuxtApp().$setStructuredData(…)` compiles without TypeScript errors.
  </done>
</task>

  <files>apps/website/app/plugins/console.client.ts</files>
  <action>
    Current file suppresses `console.debug`, `console.warn`, `console.error`, and `console.info`
    in production (i.e., when `import.meta.dev` is false). This hides real errors from developers.

    Replace the entire file contents with:

    ```typescript
    export default defineNuxtPlugin(() => {
      if (!import.meta.dev) {
        console.log = () => {};
        console.debug = () => {};

      }
    });
    ```

    Key changes:
    - Condition flipped: `if (!import.meta.dev)` instead of `if (import.meta.dev) { … } else { … }`
    - Only `console.log` and `console.debug` are suppressed
    - `console.warn`, `console.error`, `console.info` are left active (operational visibility)

    Note: The `console.log = () => {}` line was previously commented out — uncomment it (logs
    ARE noise in production and should be suppressed).
  </action>

    ```
    // Before:

    ```

    ```
      "adData",
    ```

  <name>Task 4: BUG-04 + BUG-05 — Add await + key to mis-anuncios and mis-ordenes</name>
  <files>
    apps/website/app/pages/cuenta/mis-anuncios.vue,
    apps/website/app/pages/cuenta/mis-ordenes.vue
  </files>
  <action>
    Both pages call `useAsyncData` without `await` and without a key. Without `await`, SSR completes
    before the data fetch resolves, causing hydration mismatches. Without a key, Nuxt cannot
    serialize the payload for SSR→CSR hand-off.

    **Fix 1 — apps/website/app/pages/cuenta/mis-anuncios.vue (line ~80):**
    ```typescript
    // Before:
    useAsyncData(async () => {
      const counts = await userStore.loadUserAdCounts();
      // Populate all tab counts at once from the single response
      for (const tab of tabs.value) {
        tab.count = counts[tab.value] ?? 0;
      }
      await loadAds(); // Cargar los anuncios del filtro actual
    });

    // After:
    await useAsyncData("mis-anuncios", async () => {
      const counts = await userStore.loadUserAdCounts();
      // Populate all tab counts at once from the single response
      for (const tab of tabs.value) {
        tab.count = counts[tab.value] ?? 0;
      }
      await loadAds(); // Cargar los anuncios del filtro actual
    });
    ```

    **Fix 2 — apps/website/app/pages/cuenta/mis-ordenes.vue (line ~61):**
    ```typescript
    // Before:
    useAsyncData(async () => {
      await loadOrders();
    });

    // After:
    await useAsyncData("mis-ordenes", async () => {
      await loadOrders();
    });
    ```

    Both pages are behind `middleware: "auth"` — SSR hydration correctness is critical for
    authenticated content rendering.

    Do NOT change anything else in these files (definePageMeta, layout, watchers, etc.).

    - Neither file has a bare `useAsyncData(` call without await or key
