Migrate 3 composables from `strapi.find()`/`strapi.findOne()` to `useApiClient`. These are standalone composable functions (not Pinia stores), so `useApiClient()` is called inside the function body or at composable root.

Special cases:
- `useStrapi.ts` is a thin legacy wrapper around `strapi.find("ads")` — barely used, simple migration.
- `useOrderById.ts` uses `strapi.findOne` which returns `{ data: T }` — CRITICAL: the caller `pagar/gracias.vue` destructures `const { data } = await strapi.findOne(...)`. After migration to `useApiClient`, the raw body from Strapi for a single item endpoint IS `{ data: T, meta: {} }`. So `{ data }` destructuring still works.
- `usePacksList.ts` caches packs in module-level refs with TTL. The strapi SDK wraps response in `{ data }`, so current code does `response.data as unknown as Pack[]`. After migration, `useApiClient` returns raw body which IS `{ data: Pack[], meta: {} }` for collection endpoints — so `response.data` still works.

}
