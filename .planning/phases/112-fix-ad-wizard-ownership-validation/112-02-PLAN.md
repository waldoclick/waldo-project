Add this block:
```typescript
// Ownership guard: reset wizard if draft belongs to a different user (per SEC-112-01)
if (adStore.ad.ad_id && adStore.userId !== meStore.me?.id) {
  adStore.$reset();
}

// Track current user as draft owner for future ownership checks
adStore.userId = meStore.me?.id ?? null;
```

The `adStore.userId = meStore.me?.id ?? null` line runs on every wizard entry, ensuring the userId is always set to the current user. This covers both:
- New wizard sessions (userId was null, now set)
- Returning to an existing draft (userId re-confirmed)

Note: `meStore.me` may be null at this point if the user navigated directly to /anunciar (me data loads in the useAsyncData below). However, the guard still works correctly: if `meStore.me` is null, then `meStore.me?.id` is undefined, which won't match `adStore.userId` (a number) from a previous user's session, so `$reset()` fires. The only case where `meStore.me` is null AND no reset is needed is when there is no draft (`ad_id` is null), which the `adStore.ad.ad_id` check handles.
