<objective>
Wire the `field-validation` service (plan 01) into the local registration controller so the free-text fields that exist at registration (firstname, lastname) are AI-validated before the user is created. On an explicit `false`, reject with a field-specific Spanish message; on any AI failure, registration proceeds (fail-open). Confirm the existing FormRegister.vue error handling already surfaces the message (and tweak only if needed).

Purpose: Block obviously-fake free-text input (e.g. `akhsdgKAJHSDGH` as a name) at registration without ever letting AI downtime block real users.

Output: AI gate in `registerUserLocal` validating firstname/lastname only, a Spanish field→message map, controller tests including an end-to-end fail-open proof, and verified frontend message surfacing.
</objective>
