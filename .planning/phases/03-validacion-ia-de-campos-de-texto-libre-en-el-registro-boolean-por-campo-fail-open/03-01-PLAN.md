<objective>
Create a new internal Strapi service `field-validation` that takes a map of free-text registration fields and asks the AI (via the phase-02 `ai-provider` orchestrator) to judge whether each value is plausibly real, returning a per-field boolean. The service is the structural guarantee of the fail-open requirement: it NEVER throws and NEVER blocks — it returns `true` for any field on any failure, and only a parsed explicit `false` survives.

Purpose: Isolate all AI-interaction, timeout, JSON-parsing, and fail-open logic in one tested unit so the registration controller (plan 02) becomes a trivial "find a false → reject" consumer that cannot get fail-open wrong.

Output: `apps/strapi/src/services/field-validation/` (types, config, service, index) + an additive generic `generate(prompt)` export on `ai-provider` + a full Jest service test matrix proving fail-open.
</objective>
