  <action>
    For each of the 6 controller files, apply these exact transformations:

    **find() method — 6 replacements per file:**
    BEFORE: `strapi.entityService.findMany("api::xxx.xxx", { filters, populate, start: (page - 1) * pageSize, limit: pageSize, sort: query.sort || { ... } })`
    AFTER:  `strapi.db.query("api::xxx.xxx").findMany({ where: filters, populate: query.populate || "*", offset: (page - 1) * pageSize, limit: pageSize, orderBy: query.sort || { ... } })`

    BEFORE: `strapi.entityService.count("api::xxx.xxx", { filters })`
    AFTER:  `strapi.db.query("api::xxx.xxx").count({ where: filters })`

    **findOne() method:**
    BEFORE: `strapi.entityService.findOne("api::xxx.xxx", id)`
    AFTER:  `strapi.db.query("api::xxx.xxx").findOne({ where: { id } })`

    For commune.ts and region.ts findOne, the existing code has `{ populate: ["region"] }` or `{ populate: ["communes"] }`:
    BEFORE: `strapi.entityService.findOne("api::xxx.xxx", id, { populate: [...] })`
    AFTER:  `strapi.db.query("api::xxx.xxx").findOne({ where: { id }, populate: [...] })`

    **create() method:**
    BEFORE: `strapi.entityService.create("api::xxx.xxx", { data })`
    AFTER:  `strapi.db.query("api::xxx.xxx").create({ data })`

    **update() method:**
    BEFORE: `strapi.entityService.update("api::xxx.xxx", id, { data })`
    AFTER:  `strapi.db.query("api::xxx.xxx").update({ where: { id }, data })`

    **delete() method:**
    BEFORE: `strapi.entityService.delete("api::xxx.xxx", id)`
    AFTER:  `strapi.db.query("api::xxx.xxx").delete({ where: { id } })`

    **category.ts special case:** The `adCounts()` method already uses `strapi.db.query` — do NOT modify it. Only migrate: find(), findOne(), create(), update(), delete(), and the extra `findMany` in the `all()` method at the bottom of the file.

    The UIDs per file:
    - faq.ts: `"api::faq.faq"`
    - commune.ts: `"api::commune.commune"`
    - region.ts: `"api::region.region"`
    - condition.ts: `"api::condition.condition"`
    - ad-pack.ts: `"api::ad-pack.ad-pack"`
    - category.ts: `"api::category.category"`

    Remove ALL `as unknown as Record<string, unknown>` casts that were workarounds for entityService types. The db.query API does not need them.

  <action>
    Each lifecycle file has a `beforeUpdate` hook with a single `findOne` call. Apply this transformation:

    BEFORE:
    ```typescript
    const existing = await strapi.entityService.findOne(
      "api::xxx.xxx",
      where.id,
      { fields: ["name"] }
    );
    ```

    AFTER:
    ```typescript
    const existing = await strapi.db.query("api::xxx.xxx").findOne({
      where: { id: where.id },
      select: ["name"],
    });
    ```

    CRITICAL: `fields` becomes `select` in db.query. This is a common pitfall.
